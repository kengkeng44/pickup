## A. Angle research — Drops 5-min session pacing patterns

1. **Hard countdown ring, always visible.** Drops shows a circular progress arc (green → orange) at top-center throughout the session. Users always know "how much time is left today." The ring is the primary UI chrome — more prominent than any lesson content.
2. **Zero-friction session entry.** Home screen = one large "Start" / "Play" CTA + the ring. No scrolling, no chapter grid, no node hunting. Cold-start to first question: 1 tap. Expectation set before tapping: "Your 5 minutes."
3. **Strict time budget, not question budget.** The session ends when the timer expires (even mid-card), not when a lesson finishes. This removes "how many questions are left?" anxiety entirely — users commit to time, not workload.
4. **Session-end growth metaphor.** After 5 minutes, Drops shows a "water drop fills the plant" animation — visual metaphor that something grew. Satisfying and non-punishing. Users feel the session was complete regardless of how far they got.
5. **Recharge / scarcity signal.** Free tier shows "Next drop in 8h 42m" after session ends. This creates healthy anticipation vs. guilt. Premium removes the cap. The recharge timer is the primary retention hook.

## B. Pickup 現狀 — mapped against Drops angle

1. **No session duration promise.** User taps a map paw node → lesson starts immediately with `q1/11` counter. No "~2 min" or "~3 min" estimate anywhere. First-time child learners cannot pre-commit to a time budget.
2. **Text counter, no visual arc.** LessonPage HUD shows `q{idx+1}/{total}` amber pill (14px, top-right). There is no progress bar or ring fill. At `q3/11` a child has no intuitive sense of "close to done" vs. "barely started."
3. **3-tap session entry friction.** Open app → MapPage loads (fetch async) → scroll to find current paw node → tap. No "Continue" shortcut CTA from the home screen. GrandmaRecommendCarousel is content-discovery, not "resume now."
4. **CompletePanel TIME stat is decorative.** Lesson end shows XP / ACCURACY / TIME triptych (e.g., "1:43"). Time is never compared to a target or goal. There is no "you finished in under 3 min 🎉" framing that closes the Drops-style growth loop.
5. **TasksPage daily goal wiring is partially broken.** `{ icon: '🎯', text: '正確率 ≥ 80% ...', done: false }` is hardcoded `false` — never marks done regardless of performance. From a child or parent's perspective, one of the three daily goals is permanently uncompleted.

## D. Top 3 actionable (S effort)

### D1 — Fix TasksPage 正確率任務永遠 false (P0)
**File**: `src/react-app/pages/TasksPage.tsx:46`
**Change**: Replace `done: false` with a lessons-completed proxy.
```tsx
// Before
{ icon: '🎯', text: '正確率 ≥ 80% · Accuracy ≥ 80% (待 session log)', done: false },

// After — proxy: "今日完成 2+ lessons" until session log wires accuracy
const todayLessons = (() => { let n = 0; for (let c = 1; c <= 8; c++) n += readCompletedLessons(c).size; return n; })();
{ icon: '🎯', text: `今日完成 2 個 lesson · ${todayLessons}/2 done`, done: todayLessons >= 2 },
```
This removes the "永遠未完成" perception for active users.

### D2 — Add visual progress bar to LessonPage (P1)
**File**: `src/react-app/pages/LessonPage.tsx`, after the `q-counter` pill (line ~120)
**Change**: Insert a 1-row fill bar between the top nav row and history:
```tsx
<div style={{ height: 8, background: '#f0e6d3', borderRadius: 4, margin: '0 0 12px', overflow: 'hidden' }}>

</div>
```
5 lines, zero regressions, immediate perceived-progress improvement aligned with Drops ring.

### D3 — Lesson duration estimate sub-label (P1)
**File**: `src/react-app/pages/LessonPage.tsx`, q-counter span (line ~119)

```tsx

  q{idx + 1}/{lesson.questions.length} · ~{Math.ceil(lesson.questions.length * 15 / 60)}m

```
Sets Drops-style pre-commitment ("you know how long") before the child decides to start. For 11 Q lessons this shows `~3m`.
