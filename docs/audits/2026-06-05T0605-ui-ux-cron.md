## A. Cake pattern — how it works (5 bullet)

- **Instant audio-first drop**: Cake plays a 3-5s native-speaker clip the moment you open a unit — no splash screen, no mode select, no preamble. Your brain hears English before your eyes land anywhere.
- **Scene card visual hook**: Each unit thumbnail is a screenshot from a real show — shows learner "this is real content, not a drill" within 1 glance.
- **Episode context always visible**: "S2 E4 · Scene 3" label tells you exactly where you are in a larger story arc; each scene is only 60-90s long, so "done soon" is implicit.
- **No friction to first engagement**: Cake opens to the unit list directly — tap unit → content plays instantly. No intermediary "Begin →" button.
- **Character voice = emotional hook**: You hear a character speak, not a disembodied TTS sentence. Learners attach to characters because *sound arrives before the question*.

## B. Pickup 現狀 (against this angle, 5 bullet)

- **SplashView is silent and static**: `calico-anchor.webp` at 140px, title, tagline, one green button — no audio, no mascot animation, no story preview. First impression communicates "app" not "story".
- **ChapterIntroPage doesn't auto-speak on mount**: `introZh` loads via `useEffect` fetch but never auto-plays. The "重聽介紹" button exists but it requires a tap — Cake's hook is audio you *didn't ask for* that makes you curious. Missing.
- **Map nodes carry zero story preview**: Each node shows a paw icon only. `storyBeat` labels (e.g. "春日河邊,什麼漂來?") exist in the data and aria-label but are invisible on screen. Learner has no idea what is in the next lesson before tapping.
- **SplashView footer leaks dev note in production**: `"React 18 + Vite · 60-70% 快 first load (vs Phaser 版)"` is live on the splash screen. Children and parents see this instead of any story invitation.
- **ChaptersPage is a flat text list**: 8 chapters listed as `<button>` rows with title + subtitle. No thumbnail, no "now playing" indicator, no visual cue of story genre/tone. Cake's scene grid communicates genre in under 1 second via thumbnail.

## D. Top 3 actionable (S effort)

### 1. 刪 SplashView dev footer — `src/react-app/views/SplashView.tsx:29-31`
```tsx
// DELETE this entire <p> tag:
<p style={{ fontSize: 11, color: '#8b6f4a', marginTop: 28, opacity: 0.6 }}>
  React 18 + Vite · 60-70% 快 first load (vs Phaser 版)
</p>
```
This is a dev benchmark note — it appears verbatim for children and parents. No Cake-equivalent app would show build metadata on the splash.

### 2. ChapterIntroPage: auto-speak introZh on mount — `src/react-app/pages/ChapterIntroPage.tsx`
Change the `useEffect` at line 30 to also call `speak()` after data loads:
```tsx
useEffect(() => {
  fetch(`/lessons-ch${ch}.json`)
    .then(r => r.json())
    .then((arr: Lesson[]) => {
      const first = arr[0];
      const zh = first?.intro?.zh ?? '';
      setIntroZh(zh);
      if (zh) speak(zh, 'zh-TW');   // ← add this line
    });
}, [ch]);
```
This mirrors Cake's core hook: grandma's voice starts *before* the learner taps anything.

### 3. MapPage: show storyBeat label under each node — `src/react-app/pages/MapPage.tsx`
Inside the node `<button>` render (after the `<img>` at ~line 373), add a label:
```tsx
{l.storyBeat && (
  <span style={{
    position: 'absolute', bottom: -18,
    left: '50%', transform: 'translateX(-50%)',
    fontSize: 9, fontWeight: 700, color: COLOR_TEXT_MUTED,
    whiteSpace: 'nowrap', maxWidth: 90, overflow: 'hidden',
    textOverflow: 'ellipsis', pointerEvents: 'none',
  }}>
    {l.storyBeat}
  </span>
)}
```
Cake's scene title visible on thumbnail card → learner knows what's coming → tap intent is higher.
