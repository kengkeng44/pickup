## A. Angle research — Babbel SRS patterns (5 bullet)

- **Review Manager dashboard**: Dedicated screen listing all learned words grouped by mastery stage (L1 New → L5 Mastered). Users see exactly how many words are at each stage and what's due.
- **"Due today" badge on home**: Babbel's home screen shows a persistent count of items due for spaced-repetition review. Users know before opening a session how many Qs await them.
- **Color-coded mastery tiers**: Gray = not yet reviewed, Amber = learning (L2-L3), Green = mastered (L4-L5). Every word card shows its tier at a glance — no hidden state.
- **Self-rating after reveal**: After seeing the answer, learner rates difficulty (Too Easy / Good / Too Hard). This drives next-interval calc. Babbel makes the algorithm visible via the card UI itself.
- **Intentional review entry**: Review sessions are explicitly launched ("Review 12 due words →") vs silently injected. Learner knows they're in review mode, not surprised mid-lesson.

## B. Pickup 現狀 — aligned to Babbel SRS angle (5 bullet)

- **SRS queue completely invisible**: `readSrsQueue()` in `src/data/storyKitten.ts:360` reads from `localStorage('wordwar.srs.kitten')` — that queue is never surfaced on any page (MapView, TasksPage, ProfilePage, AlertsPage). User has zero awareness of pending review items.
- **localStorage key mismatch (P0 bug)**: `storyKitten.ts:302` writes to key `'wordwar.srs.kitten'`; `LessonPage.tsx:423` reads from `'pickup.story.srs'`. These never match — `srsQueueSize` in every notif `TriggerContext` is always `0`, breaking any notification logic that gates on SRS load.
- **Binary mastery only**: SRS stores a flat array of wrong-answer question IDs. No mastery tiers (L1-L5). One correct answer removes it from queue (per `removeFromSrs`). No "answer correctly 2× to graduate" logic.
- **MapView hardcodes chapter progress**: `src/react-app/views/MapView.tsx:6` has `progress: 8, total: 24` and `progress: 0, total: 24` as constants — not wired to `readCompletedLessons()`. The progress bars are decorative, not real.
- **AlertsPage achievements always `earned: false`**: `src/react-app/pages/AlertsPage.tsx:1` hardcodes all 8 achievements as `earned: false` — no wiring to streak/XP/chapter data. Achievement wall is a static placeholder.
