## A. Angle research — Drops 5-min session pacing patterns

- **Hard session cap + radial countdown**: Drops enforces exactly 5 minutes per day (free tier). A radial ring ticks down visibly. When it hits 0, the session ends — no matter where the user is. Creates urgency, prevents binge fatigue, and builds "come back tomorrow" muscle memory.
- **Instant session entry**: Zero intro screens. User opens app → vocabulary card is already on screen. Swipe gesture begins within 1 second of launch. No "Welcome back!" overlay, no mascot card, no Next CTA gate.
- **Per-item interaction time ~4–7 seconds**: Swipe-away (recognized) or drag-to-match (uncertain). Rapid throughput. Items feel "light" — even a 40-item session feels short because each unit is sub-10 seconds.
- **Session-end celebration + scarcity signal**: When time runs out, a brief animation fires, then: "You've learned X words today. Come back tomorrow for more!" The scarcity message is the retention hook — not the celebration itself.
- **Daily quota as explicit UI element**: A progress ring on the home screen shows "2:47 remaining today". Users know at a glance how much session budget they have left before tapping in. This pre-frames the session length and reduces drop-off from perceived overload.

## B. Pickup 現狀 — 對齊 Drops session pacing lens

- **No session length signal before tapping a node**: MapPage shows paw icons and storyBeat labels, but zero time estimate. A 14-question lesson could take 3 minutes or 9 minutes; the user cannot predict. Drops solves this with the home-screen daily ring showing time remaining.
- **NarrationRenderer dwell is too slow**: `fallbackMs = text.split(/\s+/).length * 600 + 4000` (renderers.tsx:164). A 7-word sentence = 8,200 ms fallback, plus a 2,000 ms post-audio dwell (renderers.tsx:162). Worst case: 10 seconds of static screen between narration entries. Drops items never exceed ~7 seconds at normal TTS speed.
- **No daily session scarcity / "come back tomorrow" signal**: After CompletePanel fires and user taps "完成 · Continue →", they go straight back to the map with no feedback about daily goal state. Drops' most powerful retention hook ("only 5 min/day") is entirely absent.
- **TasksPage daily goal is quantitatively vague**: "完成 1 個 lesson" is the task, but no estimated time (30 sec? 5 min?). Drops' "5 min" is a concrete, memorable commitment. Users self-schedule around concrete numbers.
- **CompletePanel has no daily-goal completion moment**: The lesson-end screen shows XP / ACCURACY / TIME stats (good), but there is no celebration when today's daily goal (first lesson) is newly satisfied. Drops treats daily goal completion as a premium emotional beat.

## D. Top 3 actionable (S effort)

### D1 — Lesson time estimate on MapPage node (renderers.tsx / MapPage.tsx)
**Where**: `MapPage.tsx:349` — aria-label of each node button.
**Change**: Append ` · ~${Math.ceil(l.questions.length * 15 / 60)}min` to aria-label (and optionally to storyBeat display label).
**Why Drops lens**: Drops home ring shows "2:47 remaining today." Users who can pre-estimate 3 min vs 8 min make better commit decisions → fewer mid-lesson abandons.

### D2 — NarrationRenderer pacing fix (renderers.tsx:162,164)
**Where**: `src/react-app/renderers.tsx` line 162 (`dwellAdvance` timer) and line 164 (`fallbackMs` formula).
**Change**:
- Line 162: `window.setTimeout(advanceOnce, 2000)` → `window.setTimeout(advanceOnce, 1000)`
- Line 164: `text.split(/\s+/).length * 600 + 4000` → `text.split(/\s+/).length * 400 + 2500`
**Why Drops lens**: Drops never lets a static item sit for 10 seconds. 5–6 second max per narration line matches the "light touch, rapid rhythm" feel that makes Drops sessions feel effortless despite covering 40+ items.

### D3 — CompletePanel daily-goal banner (LessonPage.tsx)
**Where**: `src/react-app/pages/LessonPage.tsx` — `CompletePanel` component, above the Continue button.
**Change**: Before calling `markLessonCompleted`, check if any lesson was already completed today:
```ts
let wasFirstLessonToday = true;
for (let ch = 1; ch <= 8; ch++) {
  if (readCompletedLessons(ch).size > 0) { wasFirstLessonToday = false; break; }
}
```
If `wasFirstLessonToday === true`, render:
```tsx
<div style={{ padding: '8px 14px', background: '#fef3c7', border: '1.5px solid #e7a44a', borderRadius: 10, marginBottom: 12, fontWeight: 800, fontSize: 14, color: '#78350f' }}>
  🎯 今日目標達成 · Daily goal complete!
</div>
```
**Why Drops lens**: Drops' most studied retention driver is the scarcity+achievement moment at session end: "You did it for today." Without this beat, daily-habit formation has no closure signal.
