## A. Angle research (Duolingo Streak/League gamification, 2025-2026)

1. **Streak Freeze reduces churn 21%** for at-risk users. Duolingo ships both proactive (Freeze bought before miss) and reactive (Streak Repair bought after miss with gems). The reactive path is crucial: users who've broken a streak need a warm "you can fix this" moment, not a cold 0 reset.
2. **Streak Society (2025)**: milestone identity signal at 365+ days — streak counter appears *under username* on the public leaderboard, visible to every league participant. Turns a private counter into a social identity badge.
3. **Friend Streaks (Jan 2025)**: mutual consecutive-days tracking. Users with ≥1 friend streak are 22% more likely to complete daily lesson. Social accountability multiplies solo loss-aversion.
4. **Weekly Leagues (10 tiers, 7-day XP race)**: promotion/demotion stakes create asymmetric loss aversion — fear of dropping > hope of advancing. Matched by skill so leagues stay competitive without demoralizing new users. Post-lesson screen always shows leaderboard delta.
5. **Layered architecture rule**: each mechanic targets a distinct retention function (streak = daily habit / freeze = churn rescue / league = social competition / quest = goal clarity). No single feature covers all segments — together they hold 47.7M DAU and 36% YoY growth in 2026.

## B. Pickup 現狀 (對齊 Streak/League angle)

1. **✅ Streak Freeze exists** (`src/data/streak.ts`) — 2 default freezes, auto-consumed on 2+ day gap. Good Mochi-framing ("Mochi 幫你保住了"), but **no toast/banner when freeze fires** so user never knows it happened — the "reward moment" is silent.
2. **⚠️ P0 — Achievements all hardcoded `earned: false`** (`src/react-app/pages/AlertsPage.tsx:1-8`): all 8 badges show locked regardless of actual user data. Users with 7-day streaks, 1000 XP, or completed Ch1 see zero earned badges. The entire reward layer is unlit.
3. **⚠️ P1 — No streak repair CTA** when streak resets to 1 (all freezes consumed + miss). Pickup has coins currency already (`src/data/coins.ts`) but no "spend coins to repair" flow. Users who lose a long streak get no warm recovery path.
4. **⚠️ P1 — Daily task accuracy row always unchecked** (`TasksPage.tsx:~50`): `done: false` hardcoded for "正確率 ≥ 80%" — a visible lie that erodes trust in the task system.
5. **❌ No league/leaderboard layer** — Pickup is solo-only. For the 8-12 child audience, parent-child or sibling "family league" is a missed retention lever (no social pressure = churn if solo habit breaks).

## D. Top 3 actionable (S effort, ship now)

### D1 — Achievement Engine wire-up (P0, S 25min)
**File**: `src/react-app/pages/AlertsPage.tsx:1-11`

Replace hardcoded `earned: false` with runtime computation:

```tsx
import { readStreak } from '../../data/streak';
import { readXp } from '../../data/xp';
import { readCompletedLessons } from '../../store/runStore';

function computeEarned() {
  const streak = readStreak();
  const xp = readXp();
  const ch1Done = readCompletedLessons(1).size;
  const allChDone = [1,2,3,4,5,6,7,8].every(ch => readCompletedLessons(ch).size >= 24);
  return {
    1: ch1Done >= 1,           // 第一步 — completed ≥1 lesson
    2: streak >= 3,            // 三日連擊
    3: streak >= 7,            // 七日狂熱
    4: false,                  // 完美主義 — needs per-lesson accuracy log (deferred)
    5: readCompletedLessons(2).size >= 1,  // 探索者 — Ch2 unlocked
    6: ch1Done >= 24,          // 說書人 — Ch1 全章
    7: xp >= 1000,             // 神射手
    8: allChDone,              // 畢業生
  };
}
```

Then in `ACHIEVEMENTS` map: `earned: computeEarned()[a.id] ?? false`.

Impact: 7 of 8 badges can now actually light up. First reward moment users have ever seen in Achievements tab.

### D3 — Daily task accuracy wire-up (P2, S 20min)
**File**: `src/react-app/pages/TasksPage.tsx:~50`

`lessonAnswerLog` is already collected in `LessonPage` memory. Expose it via a sessionStorage key `pickup.session.accuracy` written at `_showLessonStats()`:

sessionStorage.setItem('pickup.session.accuracy', String(accuracy));

Impact: "正確率 ≥ 80%" quest row can now turn green — closes visible task system dishonesty.

**ARCH-REC #10 細節**: `AlertsPage.tsx` 現在是純靜態 JSX，8 個 `earned: false` 讓整個 achievements tab 對任何用戶都是全灰。業界標準（Duolingo、Brilliant、Khan Academy）的 achievement engine 都是 render-time 從 user state 派生 earned flag，而非 hardcoded。Pickup 的獨特優勢是所有 read APIs 已存在（`readStreak` / `readXp` / `readCompletedLessons`），只需要在 AlertsPage mount 時 call 並 compare — 約 20 行改動，零資料層變動。完成後 7/8 個 badge 立即可以 earn，achievement tab 從裝飾品變成真實 reward system。
