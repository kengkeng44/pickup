# Pickup UI/UX Cron Audit — 2026-06-01 08:31 UTC

## A. Trend research summary

- **Duolingo SDUI (Jan 2025)**: server-driven UI delivers layout + interaction logic from the backend per-screen — allows instant A/B experiment and bug fix without app release cycle. For Pickup: narration entry config (button labels, auto-advance timing) could move to JSON instead of hardcoded constants. Source: https://blog.duolingo.com/server-driven-ui/
- **Duolingo frontend prediction**: on session complete, the client immediately increments its local copy of progress + XP state — even offline — before server ack. This makes the completion feel instant. Pickup's `markLessonCompleted` is synchronous localStorage, which is fine, but coin/XP values aren't reflected in HUD post-lesson. Source: https://blog.duolingo.com/frontend-prediction/
- **Rive state machines for mascots (2025)**: an educational app (simpleclub) saw +62% DAU and +28% 30-day retention after adding a breathing idle / correct / wrong Rive mascot. Interactive mascots function as "emotional anchors that soften error states." Pickup's calico-anchor uses a CSS `pickup-mascot-bob` keyframe — static, no reactive states. Source: https://dev.to/uianimation/engineering-interactive-mascots-with-rives-state-machine-and-runtime-architecture-4e2h
- **Personalized home screen (Duolingo 2024-25)**: home redesign research showed daily-goal card + resume-lesson action in the first viewport is the highest-retention home layout. Source: https://blog.duolingo.com/new-duolingo-home-screen-design/
- **Cake / Drops pattern**: lesson entry shows a progress bar across the top of the screen (not just a text chip). Users who see a visual bar complete the lesson at higher rates than those who see only "q3/14" text. Source: https://dezyn.pro/examples/language-learning-app-ui-inspiration
