## A. Brilliant interactive demo patterns (5 bullets)

- **Curiosity hook before instruction.** Every Brilliant lesson opens with a visual or provocative question ("Why do snowflakes have 6 arms?") before any text. The hook fires curiosity before teaching, making the brain receptive. Pickup narration auto-plays immediately with no dwell — no space for anticipation.
- **"Think then reveal" — pre-commit before options appear.** Brilliant's interactive exercises often show the question for 1-2s with NO answer options visible. The user forms a prediction before choices appear, improving retrieval strength. Pickup renders options the same frame as the question text.
- **Interactive construction, not just selection.** Brilliant uses sliders, drag-and-drop, word-snap animations where each placed element has its own micro-feedback. Pickup's `TapTilesRenderer` is the closest equivalent but the "picked sentence" target zone is plain text — no snap animation, no per-word placement cue.
- **Lesson end = concept summary, not stats.** Brilliant closes every lesson with a short synthesis: "You just learned: [concept statement]." It recaps the insight, not the score. Pickup's `CompletePanel` shows XP / Accuracy / Time — three score numbers. Zero recall of what content was covered.
- **Single-word undo vs clear-all.** Brilliant's construction tasks let users un-place individual elements without resetting everything. Pickup's tap-tiles only has `清除` (clear all) — a high-anxiety reset that discourages exploration.

## B. Pickup 現狀 (對齊 Brilliant lens — 5 bullets)

- **Lesson opener = narration bubble auto-plays, zero curiosity hook.** `NarrationRenderer` mounts, calls `speak(text)` immediately, shows a speech bubble. No visual pause, no anticipation moment. Spec R1 removed the intro overlay (B.142), but didn't replace it with a curiosity hook — the lesson just starts cold.
- **Options render simultaneously with question text.** In `ListenMcRenderer` and `ListenTfRenderer`, the `<div className="pickup-answer-sticky">` and the speaker prompt render together on mount. There is no "listen window" period before options become tappable. Pre-commit dwell = 0ms.
- **TapTilesRenderer target zone is unformatted.** The "picked sentence" display (`<div style={{ minHeight: 60... }}>`) shows plain joined text: `picked.map(i => tiles[i]).join(' ')`. No per-word chip, no snap animation, no visual "slot" for each word. Tiles just disappear from pool and text accumulates — feedback is read-only, not interactive.
- **CompletePanel has XP/Accuracy/Time but no content recall.** `CompletePanel` (LessonPage.tsx:150) renders three `<Stat>` boxes. The `log` array of answered questions is available but unused for content display. User leaves the lesson knowing their score but not which sentences they learned.
- **`EmojiPickRenderer` wrong-answer = `pickup-wobble` shake only.** Wrong taps call `sfxWrong()` + add `pickup-wobble` class for 400ms. No background color change on the wrong button — visually, a wrong choice looks identical to idle after the shake settles. Brilliant's wrong-answer state uses a distinct color (red/pink) that persists until the user re-chooses.
