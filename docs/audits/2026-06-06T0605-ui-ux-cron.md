## A. Angle research — Duolingo Streak/League pattern (5 bullets)

- **Streak + Freeze shield**: Duo shows flame count + a translucent "shield" badge when a Streak Freeze is equipped. The shield is earnable (spend gems) or purchasable. Removes the primary churn moment: "I missed one day, game over." Internal data: streak freeze reduces churn -21% at day-7.
- **Daily XP goal bar** (not binary tasks): Duo replaced "complete a lesson" with a progress bar toward a daily XP quota (e.g. 10 XP = casual, 20 = regular, 50 = intense). The bar fills dynamically as you earn XP — visual momentum even from partial practice.
- **League ceremony**: Weekly leagues (Bronze → Silver → Gold → Platinum → Diamond) with animated promotion/demotion cards. Top 10 of 30 promote. The ceremony is a weekly retention trigger independent of streak. *Not appropriate for Pickup's "grandma storytelling" tone — skip.*
- **Streak Repair**: If you miss a day, a one-time "repair" option (costs gems) appears for 24h. Lowers the "I've already failed, why bother" dropout cliff.
- **Milestone celebrations**: At 7, 30, 100, 365 days Duo shows a full-screen flame celebration with confetti and a "share" CTA. Each milestone is a social amplification moment AND a natural retention checkpoint.

## B. Pickup 現狀 (對齊 Duolingo Streak/League angle — 5 bullets)

- **Streak count exists but is inert**: `src/data/streak.ts` correctly persists streak count + last-date. TasksPage renders `{streak} 天` with 🔥 emoji. But there is no freeze mechanic, no milestone ceremony, no "you're in danger" warning — streak count is read-only display only.
- **AlertsPage achievements are hardcoded `earned: false`**: `src/react-app/pages/AlertsPage.tsx` defines `ACHIEVEMENTS` with `earned: false` literals — it never calls `evaluateAchievements()` from `src/data/achievements.ts`. Result: all 8 achievement tiles are permanently greyed-out/locked regardless of real state. The unlock logic exists but is silently disconnected.
- **TasksPage uses binary done/○ not an XP progress bar**: Three tasks render `✓` or `○`. The 3rd task ("正確率 ≥ 80%") is permanently `done: false` with a TODO comment. No incremental progress bar fills as XP accumulates during the session — the only momentum signal is the Streak hero card at top.
- **Streak stored in both Zustand and localStorage simultaneously but diverges**: `runStore.ts` has `streak` in state (set via `useRunStore`) while `streak.ts` has `readStreak()` / `updateStreak()` reading from localStorage directly. ProfilePage reads from runStore via `useRunStore`; streak display in the old Phaser HUD reads via `readStreak()`. Two sources of truth risk silent mismatch.
- **No Streak Freeze, no streak repair, no milestone ceremony**: The master matrix notes Streak Freeze as #2 priority (churn -21%), but no implementation exists. Missing: freeze token UI, storage key, daily-check bypass logic, milestone celebration screens at 7/30-day.

## D. Top 3 actionable (S effort, high impact)

### D1 — Fix AlertsPage: wire to `evaluateAchievements()` (P0, XS)
**File**: `src/react-app/pages/AlertsPage.tsx`
**Problem**: All 8 badges permanently locked — streak-3/xp-50/ch1-complete achievements earned by real users are invisible.
**Fix**: Import `evaluateAchievements` from `../../data/achievements`, call inside component, map to render grid. Delete the hardcoded `ACHIEVEMENTS` const.
**Impact**: Immediate — users who have earned badges will see them for the first time. Trust in gamification system restored.

### D2 — Daily XP progress bar in TasksPage (P1, S)
**File**: `src/react-app/pages/TasksPage.tsx`
**Problem**: Binary ✓/○ gives no momentum when partially done. The "80% accuracy" task is hardcoded-false forever.
**Fix**: Replace 3-row task list with: (a) streak hero card (keep), (b) daily XP progress bar: `readXp() today vs goal` — requires a `readTodayXp()` function that stores `pickup.xp.today` + reset date, (c) remove the dead 80%-accuracy row.
**Impact**: Every correct answer fills a visible bar — reinforces the session reward loop.

### D3 — Streak dual-source unification (P1, S)
**File**: `src/store/runStore.ts` + `src/react-app/pages/TasksPage.tsx` + `ProfilePage.tsx`
**Problem**: `runStore.streak` (Zustand) and `readStreak()` (localStorage) can diverge. ProfilePage reads runStore; Phaser HUD reads localStorage. Silent inconsistency on app load.
**Fix**: In App.tsx boot effect, call `readStreak()` and sync into runStore via a new `setStreak(n)` action. All reads go through runStore. Remove redundant dual-writes.
**Impact**: Eliminates "streak shows 5 in profile but 0 in HUD" bug class.
