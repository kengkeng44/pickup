## A. Angle research — Duolingo Stories patterns (5 bullets)

- **Character-assigned speech bubbles with positional alignment**: each character owns a screen side — "grandma/NPC" on left, "player perspective" or alternate voice on right. Bubbles have a CSS tail (triangle notch) pointing toward the circular avatar to create spatial dialogue feel.
- **Per-character avatar shown inline with each bubble**: a 48–56px circular avatar of the speaking character appears beside each dialogue line; avatar changes between grandma / Mochi / Hana visually anchoring WHO is speaking — not just a pill label.
- **Manual Continue tap (not auto-advance)**: Duolingo Stories requires explicit user tap to advance each narration line. This is intentional — gives learner control of pacing, reduces anxiety, and makes re-listening natural (tap speaker, then tap Continue on own terms).
- **Karaoke-style in-progress word highlight**: as TTS plays, the word currently being spoken gets a subtle amber/yellow highlight — trains phonics + prevents zoning out. Requires per-word timing data or estimated character-offset sync.
- **Character emotion states**: avatar image swaps between "neutral", "happy", "surprised", "thinking" expressions during key story beats — emotionally anchors comprehension without extra text.

## B. Pickup 現狀 — mapped against this angle (5 bullets)

- **All NarrationRenderer bubbles use hardcoded `calico-anchor.webp`** regardless of `q.speaker`. When grandma is speaking, her avatar is still the calico cat. SPEAKER_META has the correct emoji+label for grandma/hana/narrator but the actual avatar `<img>` is hardcoded. `src/react-app/renderers.tsx:185`.
- **No left/right speaker-side alignment**: every narration bubble uses the same `flex-direction: row` (avatar left, text right) regardless of whether the speaker is grandma, Mochi, or narrator. Dialogue feel absent — feels monolithic.
- **SpeakerBadge is a small pill chip, not a visual anchor**: `{ display: 'inline-flex', padding: '3px 9px', fontSize: 11 }` — occupies ~24px height above the bubble. Duolingo Stories uses the avatar AS the spatial anchor, not a text label. The pill is redundant cognitive load if the avatar is character-correct.
- **NarrationRenderer auto-advances after TTS ends + 2s dwell** (no visible Continue button): spec R2 mentions a "繼續 ↓ Continue green button" but current implementation fires `onAdvance` automatically. User has no tactile confirmation they've "received" each line. Senior/child learners may feel rushed.
- **No karaoke highlight, no character emotion states**: `calico-anchor.webp` is a single static image. No word-by-word timing highlight during playback. Both are Duolingo Stories signature mechanics that Pickup's grandma storytelling framework would benefit from most.

## D. Top 3 actionable (S effort, 可立刻 ship)

### 1. Per-speaker avatar map in NarrationRenderer
**位置**: `src/react-app/renderers.tsx` ~L133（SPEAKER_META 附近）

```ts
const SPEAKER_AVATAR: Record<string, string> = {
  mochi:    '/mascots/calico-anchor.webp',
  grandma:  '/mascots/iso-grandma.webp',
  hana:     '/mascots/iso-shiba.webp',
  narrator: '/mascots/icon-book.webp',  // fallback: icon-speaker
};
```

在 `NarrationRenderer` 的 `<img>` 把 `src="/mascots/calico-anchor.webp"` 改為 `src={SPEAKER_AVATAR[q.speaker ?? 'narrator'] ?? '/mascots/calico-anchor.webp'}`。

**影響**: 奶奶說話 → 看到奶奶臉。品牌連貫性 + 兒童/家長一眼知道誰在說話。

---

### 2. Speaker-side alignment (dialogue positioning)
**位置**: `src/react-app/renderers.tsx` ~L182（NarrationRenderer return 的 `<div style={{display:'flex'...}}>` ）

```ts
const isRight = q.speaker === 'mochi' || q.speaker === 'hana';
// wrapper flex-direction
flexDirection: isRight ? 'row-reverse' : 'row'
// bubble borderRadius
borderRadius: isRight ? '14px 4px 14px 14px' : '4px 14px 14px 14px'
```

**影響**: grandma 在左說故事，Mochi/Hana 在右回應 — Duolingo Stories 對話空間感，視覺 variety 防止 fatigue。

### 3. Visible Continue button on narration (tappable, auto-advance fallback)
**位置**: `src/react-app/renderers.tsx` NarrationRenderer return，bubble `</div>` 後加：

```tsx
<button
  onClick={() => { if (!advancedRef.current) { advancedRef.current = true; onAdvance(text); }}}
  style={{
    marginTop: 12, width: '100%', padding: '12px 0',
    background: '#7ac74a', color: '#fff',
    border: 'none', borderBottom: '3px solid #5d9a35', borderRadius: 12,
    fontSize: 15, fontWeight: 900, cursor: 'pointer', fontFamily: 'inherit',
  }}
>繼續 · Continue ↓</button>
```

Auto-advance fallback timer 從 7s → 15s (讓兒童/老人自己掌控節奏)。
**影響**: 符合 Duolingo Stories 明確推進感，兒童/銀髮用戶不被 auto-advance 嚇到。
