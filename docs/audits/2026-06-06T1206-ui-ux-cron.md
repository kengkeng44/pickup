## A. Angle research — Anki SRS design patterns (5 bullets)

- **Interval scheduling with ease factor**: Each card stores a numeric ease factor (starting 2.5). Wrong = interval resets to 1min/10min learning steps; correct = interval × ease (days). This guarantees cards are never over-reviewed AND never forgotten past a known horizon.
- **4-grade self-evaluation (Again / Hard / Good / Easy)**: After the answer reveals, the user declares how hard it felt. This single UX choice feeds the SM-2 algorithm — it is Anki's core product surface, not just a settings toggle.
- **Due-today queue as the daily ritual**: Anki's home screen shows exactly one number: "X cards due." Every session starts from this queue. The contract is "do your reviews today, forgetting is prevented." The ritual makes the app open itself.
- **Visual card state (New / Learning / Review / Mature)**: Cards display their state in the browser. "Mature" (interval > 21 days) is the visible evidence of mastery — the user can see vocabulary moving from New → Young → Mature over weeks.
- **Leech detection + suspension**: A card answered wrong ≥ 8 times (leech threshold) is auto-suspended and flagged. Prevents futile grinding on cards the learner genuinely cannot retain yet; surfaces them in a separate tag for re-study strategy.

## B. Pickup 現狀 — through Anki SRS lens (5 bullets)

- **SRS = flat FIFO queue, no scheduling**: `storyKitten.ts` `LS_SRS_QUEUE` stores a plain `string[]` of wrong question IDs. `addToSrs` appends; `removeFromSrs` deletes on first correct. There is no timestamp, no interval, no ease factor. A word answered wrong today and a word answered wrong 3 weeks ago are treated identically.
- **v2.0 LessonPage has ZERO SRS integration**: `grep -rn 'srs\|readSrsQueue\|addToSrs' src/react-app/` returns empty. The active development track (`LessonPage.tsx`, `lessons-ch*.json`, `CompletePanel`) does not write to or read from the SRS queue — wrong answers in v2.0 sessions vanish with no review scheduling. The SRS that exists lives only in the deprecated `PlayScene` / `runStore` story-mode path.
- **Graduate rule = "correct once" (aggressive)**: `removeFromSrs(round.id)` fires on first correct in any context. SM-2 baseline: a card needs stable performance over multiple sessions before its interval grows. One correct answer proving memory is a known weak signal (next-day forgetting rate ≈ 50-70% for single-trial recall without spacing).
- **No SRS visibility anywhere in the UI**: Profile page shows streak/XP/coins/level. MapPage shows lesson nodes. Neither surface shows "N words in review queue," "next review due," or "mature vocabulary count." The SRS system is completely invisible to the player — they cannot feel it working.
- **SRS review is buried at chapter start, unlabeled**: The 3 SRS questions prepended to `pool` in `startRound()` carry no visual distinction from new questions. The player doesn't know they're reviewing past mistakes; they get no "🔁 Reviewing from last time" cue. Without salience, the review feels like a random question, not a callback.

## D. Top 3 actionable (S effort)

### 1. Wire v2.0 LessonPage wrong answers → SRS (P0)

In `src/react-app/pages/LessonPage.tsx` `CompletePanel` `useEffect`, add 5 lines after `markLessonCompleted`:

```ts
import { addToSrs } from '../../../data/storyKitten';
// inside CompletePanel useEffect:
log.filter(a => !a.isCorrect).forEach(a => { try { addToSrs(a.q.id); } catch {} });
```

This makes v2.0's 1100+ lesson questions feed the existing SRS queue. Zero architectural change, instant value.

### 2. "N 個複習等你" pill on MapPage (P1)

In `src/react-app/pages/MapPage.tsx`, above the chapter/lesson node list, read `readSrsQueue().length` and conditionally render:

```tsx
{srsCount > 0 && (
  <div style={{ background: '#fef3c7', border: '2px solid #e7a44a', borderRadius: 999, padding: '6px 16px', fontSize: 13, fontWeight: 800, color: '#78350f', display: 'inline-flex', gap: 6, alignItems: 'center' }}>
    🔁 {srsCount} 個複習等你 · Review ready
  </div>
)}
```

Gives players a daily ritual entry point equivalent to Anki's due-count home screen badge.

### 3. "🔁 N 個加入複習" in CompletePanel stats (P2, ~20 min)

After fixing P0 (wrong answers written to SRS), add a sub-line under the Accuracy stat:

```tsx
{log.filter(a => !a.isCorrect).length > 0 && (
  <div style={{ fontSize: 12, color: '#8b6f4a', marginTop: 4 }}>
    🔁 {log.filter(a => !a.isCorrect).length} 個加入複習庫
  </div>
)}
```

Closes the feedback loop: player sees *what* entered their review queue, making the SRS system visible for the first time.
