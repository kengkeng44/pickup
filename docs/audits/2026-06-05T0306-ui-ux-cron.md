## A. Angle research — Duolingo Stories core patterns (5 bullets)

- **Per-character speech bubbles**: Each speaker gets a unique avatar circle + bubble colour. Lin (blue), narrator (cream), learner (right-aligned). The visual identity encodes WHO is speaking faster than any text label.
- **Sequential slide-in**: Each new line appears with a 120-200ms fade-up. The active line is visually prominent; history above fades to ~60% opacity — continuous scroll never clears past entries.
- **Avatar BESIDE the bubble row, not above**: A 32-36px circular character face is flush-left next to the bubble. This makes character identity legible in peripheral vision while reading.
- **Minimal overhead per bubble**: No label pills, no borders, no explanation blocks — just the avatar, the coloured bubble, and a small speaker icon. Visual noise < Pickup's current cards.
- **Completion article as story reward**: On lesson end, ALL narration entries concatenate into a clean paragraph. Player reads (or listens to) the full story they just unlocked — powerful "earned it" feel.

## B. Pickup 現狀 (對齊 Duolingo Stories angle)

- **B1 — 全 77 narration entries in Ch1 are `speaker:'narrator'`**: Data layer never tags Grandma, Mochi, or Hana. The `SpeakerBadge` component always renders `📖 背景` regardless of content.
- **B2 — NarrationRenderer hardcodes `calico-anchor.webp`**: Line 174 `renderers.tsx`. Even if the data were fixed, the avatar never changes per speaker — Grandma's lines still show the stray cat face.
- **B3 — SpeakerBadge is a pill ABOVE the bubble**: An 11px text pill (`mochi / grandma / hana / 背景`) sits above the cream card. Duolingo puts the avatar INSIDE the bubble row, inline with the text — more immediate emotional signal.
- **B4 — History (NarrativeLine) loses all speaker context**: `LessonPage.tsx:111-126` — past entries render as plain text + a speaker icon with zero character attribution. You can't scroll back and know who said what.
- **B5 — No completion article in React LessonPage**: `CompletePanel` shows XP/Accuracy/Time stats but skips the story-paragraph reward (canonical spec R5). The spec is only partially implemented in the defunct Phaser `LessonScene.ts`.

## D. Top 3 actionable (S effort)

### S1 — Speaker-aware avatar in NarrationRenderer (`renderers.tsx:174`)

```ts
// 加在 SPEAKER_META 下方 (~line 129)
const SPEAKER_AVATAR: Record<string, string> = {
  mochi:    '/mascots/calico-anchor.webp',
  grandma:  '/mascots/iso-grandma.webp',
  hana:     '/mascots/iso-shiba.webp',
  narrator: '/mascots/calico-anchor.webp',
};
```

Then in `NarrationRenderer` line 174:
```ts
- <img src="/mascots/calico-anchor.webp" width={44} height={44} alt="" style={{ borderRadius: '50%' }} />
+ <img src={SPEAKER_AVATAR[q.speaker ?? 'narrator']} width={44} height={44} alt={q.speaker ?? ''} style={{ borderRadius: '50%' }} />
```

Same 1-line fix for `EmojiPickRenderer` (lines 553, 568).
**Impact**: 一旦 Ch1 JSON speaker 標注修好，Grandma 的台詞立即顯示奶奶臉，character identity 成立。

### S2 — Per-character bubble tint (`renderers.tsx:173`)

```ts
const SPEAKER_BUBBLE_BG: Record<string, string> = {
  mochi:    '#fff0e0',   // 暖橙米
  grandma:  '#fffaec',   // 溫黃
  hana:     '#f4ece2',   // 淺棕

```

Line 173 `background: '#fff7e8'` → `background: SPEAKER_BUBBLE_BG[q.speaker ?? 'narrator']`

**Impact**: 在 history scroll 中能瞬間用顏色辨認說話者，不需看 label。

### S3 — Carry speaker through NarrativeLine history (`LessonPage.tsx`)

```ts
// onAdvance 加第二參數
const onAdvance = (snapshot?: string, speaker?: string) => {
  if (snapshot) setHistory(h => [...h, { text: snapshot, speaker }]);
  setIdx(i => i + 1);
};
// history 改 { text, speaker }[]，NarrativeLine 加 speaker? prop
// NarrativeLine border-left: 3px solid SPEAKER_BUBBLE_BG color
```

**Impact**: 滾動歷史不再是匿名文字流，玩家可快速回溯是誰說了什麼。
