## A. Cake pattern research (5 bullets)

- **Clip-first entry**: Every lesson opens by immediately auto-playing a 3–15s native audio/video clip — no preamble, no instruction overlay, no "Lesson N" splash. The content IS the hook.
- **Ambient preview on chapter scroll**: On mobile, pausing on a chapter card triggers a 1–3s audio snippet (or animated GIF loop on cover art) — users hear the language before committing to tap.
- **Session "snacking"**: After N clips (default 5), Cake surfaces a warm "오늘 5개 완료! 🎉" exit ramp — allows guilt-free stop, naturally paces daily habit without streak threat.
- **"Heard in…" authenticity badge**: Each clip carries context ("From Friends / 오징어게임 / MrBeast") so learners feel they're engaging with real-world material, not synthetic drill. Creates aspiration pull.
- **Single CTA after clip**: Post-clip screen has exactly one forward action (Shadow / Continue) — never 3+ choices. Attention is already spent on listening; cognitive load must drop to near zero for the next step.

- **Map nodes are purely visual / silent**: 24 paw icons on MapPage — no audio preview on hover or long-press. A new user landing on an empty map hears nothing from the product until they complete the click-to-lesson chain.

### D1 — ChapterIntroPage: auto-speak intro on mount `[XS]`
**File**: `src/react-app/pages/ChapterIntroPage.tsx`
**Where**: After the `useEffect` that fetches and sets `introZh` (~line 42), add a second `useEffect`:
```tsx
useEffect(() => {
  if (introZh) speak(introZh, 'zh-TW');
}, [introZh]);
```
**Why Cake-relevant**: Cake's core promise is audio-before-text. The ChapterIntro is the closest Pickup surface to a "chapter cover clip" — it already has `speak` imported, `introZh` loaded from JSON, and a narration card UI. The single missing wire is auto-fire on mount. A child tapping a chapter name should *hear* grandma's voice within 1 second of the screen appearing, not need to find a speaker button.

### D2 — MapPage cold-start welcome audio `[S]`
**File**: `src/react-app/pages/MapPage.tsx`
**Where**: In the `useEffect` block that handles `visitToast`, or a new `useEffect` gated on `firstTime && !loading`:
```tsx
useEffect(() => {
  if (firstTime && !loading) {
    const t = window.setTimeout(
      () => speak('奶奶在院子裡等著說故事，你來了。', 'zh-TW'),
      900
    );
    return () => window.clearTimeout(t);
  }
}, [firstTime, loading]);
```
**Why Cake-relevant**: Cake's first-open plays a clip automatically within 2s. A brand-new Pickup user with xp=0 currently experiences full silence — the most critical 5-second impression window is wasted. This single `speak()` call transforms the cold-start into a sensory hello from grandma at zero visual-layout cost.
