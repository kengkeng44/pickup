## A. Angle Research — Brilliant's core UX patterns

- **Scaffolded concept progression**: Each screen introduces one concept; next screen immediately applies it. No cold questions. "See it → interact with it → answer about it" is the invariant flow.
- **Visual before verbal**: Concepts shown as animated diagram or interactive widget *before* any text question. The "aha" is experiential first, labeled second. Brilliant never drops you into a question without prior visual context.
- **Instant contextual feedback**: Wrong answer shows WHY visually — the model changes to reflect the error, not just a flash+retry. Correct explanations highlight the exact element that made the answer right.
- **Labeled concept map**: Course sidebar shows exactly what concept each node covers, which are complete, locked, and next. Each node = one concept name (not just a number or generic icon).
- **Micro-step scaffolding**: Hard interactions are broken into 2-3 warm-up sub-steps. Never asks a cognitive leap too large. 2-strike help appears progressively (hint → reveal) with escalating specificity.

## B. Pickup 現狀 (through Brilliant lens)

- **Map nodes are opaque**: All 7 lesson nodes in Ch1 render identically (paw icon, same size, same color when unlocked). Tapping a node goes directly to lesson with zero preview. `storyBeat` exists in the data (`MapPage.tsx:456` uses it for `aria-label` only — screen-reader accessible but visually invisible). Brilliant always previews: concept + type + estimated Q count before you enter.

- **No lesson intro screen**: `lesson.intro` field exists in schema but the intro overlay was deprecated at B.142 ("intro overlay deprecated (R1 spec)"). First question in a lesson can be any type (narration, emoji-pick, listen-mc) with no warm-up signal. Brilliant's pattern: first interaction is always a simpler "observation" step before the scored question.

- **NarrationRenderer has no manual skip**: `renderers.tsx:160-201` — confident learners and fast-advancing kids have no way to skip the 7s+ auto-advance. The only interaction is the replay SpeakerBtn. Fallback timer = `Math.max(7000, words * 600 + 4000)` (line 177). Brilliant always shows a "Got it →" button alongside the auto-timer.

- **ExplanationZh is text-only and disconnected from the sentence**: `renderers.tsx:129-133` — amber left-border card, appears below the question block. Does NOT highlight which word in the sentence is the subject of the explanation. For listen-mc and cloze, this means learner reads "這句話..." explanation but can't see which blank it refers to. Brilliant wires explanation back into the visual model.

- **segmentType is invisible to the learner**: `Lesson.segmentType` (e.g. `main-story`) is loaded by MapPage and LessonPage (`MapPage.tsx:30`, `LessonPage.tsx:39`) but never rendered. The q-counter chip (`LessonPage.tsx:121-123`) shows only `q{idx+1}/{total}` — no breadcrumb showing "you are in: 主故事". Brilliant's concept breadcrumb is a core navigation anchor.

## D. Top 3 Actionable (S effort)

### #1 NarrationRenderer — 加「繼續 →」skip button (`renderers.tsx:189-201`)

**Brilliant pattern**: every auto-advancing observation screen has a "Got it →" tap. Kids shouldn't wait 7s to do nothing.

```tsx
// 在 NarrationRenderer return 的 div 末尾加:
<button
  onClick={() => {
    window.clearTimeout(timer);  // cancel fallback
    onAdvance(text);
  }}
  style={{ marginTop: 12, padding: '10px 20px', background: 'transparent',
    border: '2px dashed #c8a878', borderRadius: 10, color: '#8b6f4a',
    fontSize: 14, fontWeight: 800, cursor: 'pointer', width: '100%' }}
>
  繼續 · Continue →
</button>
```

Note: `timer` ref must be lifted to outer scope (currently inline at line 178). Pattern already present in `ListenTfRenderer:220` which correctly lifts `innerTimer`.

### #3 Map node lesson preview tooltip (`MapPage.tsx:450-488`)

**Brilliant pattern**: hover/tap node shows concept name + type + Q count before entering.

```tsx
// Add state at MapPage component top:
const [previewLesson, setPreviewLesson] = useState<Lesson | null>(null);

// In node button onClick (line 457):
onClick={() => {
  if (!unlocked) return;
  if (previewLesson?.id === l.id) {
    navigate(`/lesson/${chapter}/${l.id}`);
  } else {
    setPreviewLesson(l);
    window.setTimeout(() => navigate(`/lesson/${chapter}/${l.id}`), 600);
  }
}}

// Add tooltip overlay (sibling of node button in map container):
{previewLesson && (
  <div className="pickup-fade-up" style={{
    position: 'absolute',
    left: /* node leftPx */ + NODE_SIZE + 8,
    top: /* node slot.top */,
    background: '#fff7e8', border: '2px solid #e7a44a',
    borderRadius: 10, padding: '8px 12px',
    fontSize: 13, fontWeight: 700, color: '#3c2a1c',
    zIndex: 10, maxWidth: 140, pointerEvents: 'none',
  }}>
    {previewLesson.storyBeat ?? `Lesson ${previewLesson.lessonInChapter}`}
    <div style={{ fontSize: 11, color: '#8b6f4a', marginTop: 2 }}>
      {previewLesson.questions.length}Q · {SEG_LABEL[previewLesson.segmentType] ?? ''}
    </div>
  </div>
)}
```

Tap once = preview (600ms), tap again = navigate (or let 600ms elapse). Works for both impatient kids and exploratory parents.
