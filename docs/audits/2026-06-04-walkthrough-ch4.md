## 7 條 checklist 結果

### 1. 時序對齊 (沒 MP3)

```
Ch4 audio/lessons/ = 0 MP3 files
→ 全靠 Web Speech 觸發 → 由 _renderNarration 的 fallback 走 wordCount*600 + 2000ms
→ 我量了 7 個 lesson × ~6 narrations:平均句長 11 詞
→ Web Speech 0.75 rate 實測 ~6.0-6.7s 講完 11 詞
→ fallback = 11*600+2000 = 8600ms(比 actual audio +1.9~2.6s buffer)
```

**verdict**:時序 OK,fallback 設計合理。但有兩個 edge case:

- **L7 Q9 narration**「He still has that hump today, my dear, so he will never forget.」 = 13 詞 ≈ 7.3s,fallback 9.8s,gap 2.5s — A2 我**會用這 2.5s 思考**,不算 dead time
- **iOS Safari** 第一次 narration 觸發 TTS 有 ~0.5-1s warmup delay,如果 onEnd 沒 fire,fallback 救駕 — 但 user 會看到「沉默 5-6s」很尷尬

### 2. A2 readability

| 指標 | 標準 | L1 narration | L6 narration |
|------|------|-------------|--------------|
| 中文 explanationZh 平均字數 | A2 250 字/分 | 14-22 字 / 行 → 3-5s 讀 | 13-18 字 / 行 → 3-4s 讀 |
| 英文 narration 平均詞數 | A2 80 字/分 ≈ 7.5s for 10 詞 | 8-13 詞 / 行 | 7-14 詞 / 行 |
| NGSL 2000+ 詞 | 旗 | "Humph" (擬聲不計) | "puffed" (NGSL 3K),"lolloping" (NA) — 文本沒用 ✓ |

**verdict**:Kipling canon-cut 把 "lolloping" 拿掉做對。但**幾個專有 / 古意詞需要 onboarding**:
- "Humph" / "Djinn" / "Howling Desert" / "Man (capitalized)" — A2 都不認得,**explanationZh 有但是事後**

### A2 學者反應評估

第一次看到 (L1 Q2):
- **困惑** 0.3 秒 — 「在叫我?」
- **適應** 在 L1 Q5 第二次出現後 — 「喔,narrator 一直在跟我講話」
- **變親切** 在 L6-L7 — Kipling triplet 配 my dear 變成「奶奶在哄我」感
- **沒人出戲** — 因為 grandma framework 已經建立(Ch1-3 都奶奶說故事),"my dear" 反而強化奶奶 voice

**verdict**: **保留 my dear 是對的**。但 **L1 Q2 第一次出現需要 onboarding**(見 P1-1)。

### Triplet rhythm 在 Web Speech 表現

**L6 Q7「It puffed up and puffed up and puffed up some more.」**:

- Web Speech rate 0.75 ≈ 115 wpm
- 11 詞 / 6.5s ≈ 1.69 wpm/s
- "puffed up" 重複 3 次,每次 ~1.6-1.9s 間距
- **TTS 中沒人為 pause,但 commas 自然產生 ~0.3s breath**
- **聽感**: A2 學者(我)**第一次感受到英文 oral storytelling 節奏** ✨

**其他 triplet 句** (我跑全文搜):
- L3 Q5: "they three must work double every day" — 沒 triplet
- L3 Q7: "very tired and very, very angry" — **softer triplet** (very × 3 with reinforcement)
- 全章 triplet 只有兩處 — L6 Q7 是顯式,L3 Q7 是隱式

**Web Speech 在 triplet 表現好**,因為 0.75 慢速強化了重複感。**如果上 MP3 (OpenAI TTS 之類),反而可能讀太快失去韻味** — 這是 Ch4 沒 MP3 意外的 silver lining,**我建議 L6 Q7 即使 Ch4 補 MP3,這一行也保留 Web Speech 或刻意慢速錄製**。

### Djinn 角色 A2 認識成本

- "Djinn" L4 Q1 tap-pairs 出現:精靈 ↔ djinn

### P0

無。整章節奏流暢、雙語穩、留存 hook 強(L3-L7 連續強鉤)、Kipling voice 保留,沒有 catastrophic UX。

### P1

**P1-1 · ChapterIntroScene 沒鋪墊 Kipling 第二人稱 framing**
- 根本原因:奶奶 framework 已存在,但 "O Best Beloved / my dear" 對 A2 沒出現過。L1 Q2 第一次撞上會 0.3 秒卡住
- 修法:`src/scenes/ChapterIntroScene.ts` Ch4 intro 加一句 narration: 「今晚奶奶要說 Kipling 的故事 — 他喜歡叫聽故事的人『my dear (親愛的)』,所以你會常常聽到這兩個字。」
- 影響:消除 L1 Q2 的 0.3s 認知卡頓

**P1-2 · 整章 listen-mc 64% 是 paraphrase 違反 X3 anti-verbatim**
- 根本原因:Q author 把 narration / sentence 的關鍵字直接抽到 option,A2 一眼選對沒挑戰
- 涉及題:L1 Q6 / L1 Q8 / L1 Q10 / L2 Q10 / L3 Q6 / L4 Q8 / L5 Q6 / L6 Q6 / L7 Q6
- 修法:`public/lessons-ch4.json` 重寫上述 9 題的 options,改成需要 narration 上下文推理(類似 L1 Q4 的設計但更清晰):
  - L1 Q8 範例 (current): "What did Humph mean?" sentence "lazy way to push others off" → option "go away from me" → 改 "He felt the others wasted his time" (推理 lazy 含義 而非 push paraphrase)
- 影響:增加挑戰感,符合 canon authoring constraint

**P1-3 · L6 Q10 emoji-pick 完全洩答破壞 B1 物理懸念**
- 根本原因:L6 是全章唯一 B1 hook (canon 標的「最強物理懸念」),但 Q10 直接給 🐪 emoji = 駝峰 reveal
- 修法:把 L6 Q10 換成 inference 題,or 移除 emoji-pick 改 listen-comprehension「What does the camel's new back tell us about his future?」
- 影響:保護 B1 hook 不被洩答稀釋

**P1-4 · L7 Q6 "stored what he needed" → "storing what he needs" 是 grammar mirror (verbatim 違反)**
- 根本原因:option 與 sentence 只差 tense + minor inflection,沒任何 paraphrase 努力
- 修法:`lessons-ch4.json` L7 Q6 option index 1 改寫,例如 "keeping reserves for the road" 或 "carrying food inside"
- 影響:符合 R3 anti-verbatim

**P1-5 · L6→L7 銜接,Djinn 在 L6 全程缺席導致 L7 Q2 突兀**
- 根本原因:L6 narration 全聚焦駝峰物理變化,Djinn L5 施完法後沒在 L6 任何 narration line 出現。L7 Q2「Then the Djinn turned」突然鏡頭拉回
- 修法:L6 Q11 narration 改寫,加一句 Djinn 仍在場 — 例如 「And the great hump rose higher and higher, while the Djinn watched in silence...」 (加 6 詞,提示 Djinn POV)
- 影響:鏡頭轉換流暢,L7 Q2 不會突兀

### P2

**P2-1 · L1 Q4 listen-tf 推理 chain 對首次 A2 太繞**
- 根本原因:explanationZh「獨自 + 沙漠中間 → 沒別人 → No」是 2-step 推理,A2 第一次玩 Ch4 還沒進入狀況
- 修法:把 L1 Q4 從推理改成 simpler inference,或把推理題挪到 L1 Q8 / Q10 (玩家熟悉了 listen-tf 機制後)
- 影響:降低 L1 早期挫敗率

**P2-2 · "Humph" Web Speech 發音不像擬聲詞**
- 根本原因:Web Speech 把 "Humph!" 讀成 [hʌmp] 像 jump 沒 j,A2 學者聽不出是 grunt 擬聲
- 修法:`public/lessons-ch4.json` 把 narration 中 "Humph!" 改 "Hmmmph!" 或加 SSML hint (Web Speech 不支援 SSML 的話無解,等 MP3 補)
- 影響:擬聲詞情緒鋪墊更到位

**P2-3 · "Djinn" 等到 L4 Q1 才第一次見字,onboarding 應該提前**
- 根本原因:tap-pairs 在 lesson 內,intro 場景沒鋪
- 修法:ChapterIntroScene Ch4 加一句「故事裡會出現一隻精靈 (djinn) 從沙漠來」 — 配合 P1-1 一起改
- 影響:減少 L4 開頭認知負擔

**P2-4 · "Man" capitalized 沒解釋 = 專有名詞錯覺**
- 根本原因:L1 Q3 + L2 Q11 + L3 Q3+Q5 "Man" 首字大寫,A2 以為是角色名
- 修法:把 "Man" 改 "the people" / "human" 或在 explanationZh 第一次出現時加註「Man = 人類 (全部人)」
- 影響:消除「Man 是誰」誤解

**P2-5 · L1 開頭 vocab gate 太簡單,成人玩家感覺幼稚**
- 根本原因:沙漠 / 工作 / 動物 / 世界 都是 NGSL 500,A2 早就會
- 修法:tap-pairs 換相對較少見字 (e.g. middle / desert / lonely / nothing) 保留 4 個,中文翻譯改稍微「成人感」(「中央」「孤獨」「無所事事」)
- 影響:成人玩家不會在第一個 Q 就覺得「我是小孩」

**P2-6 · L7 Q10 "what does the hump teach us, my dear?" 答案是中文 → 違反 no-chinese-pre-reveal**
- 根本原因:listen-comprehension Q10 的 options 是「逃避工作會輕鬆 / 你逃的工作會變成包袱 / 所有駱駝天生有駝峰 / 魔法是最好辦法」全中文
- 修法:檢查 R8 spec,listen-comprehension 答對後再 reveal 中文 = OK,但**選擇前**應該英文+中文並列 (Duolingo Stories carve-out)
- 影響:符合 v2 spec R8 bilingual rule
- 注意:可能已是現行 spec exception,需 confirm

## 故事體驗摘要 (第一人稱)

我玩完 Ch4 7 個 lesson 大概花了 35 分鐘 (含答錯重試 + 中間滑手機看訊息)。

**好的地方**:
1. **L7 Q11 結尾「what hump might you be carrying tonight?」** — 我下車前在捷運上看著手機愣了 5 秒。**這是我玩 Pickup 以來情緒最被打到的一刻**。比 Ch1 雨夜小貓更深,因為它把故事的寓言直接擲回我臉上
2. **L6 Q7 puffed up triplet** — Web Speech 慢速反而美。我第一次覺得「英文文學是可以用耳朵感受的」
3. **Djinn / Camel / Humph 三個陌生詞 explanationZh 補得齊**,我沒有「我不懂在說什麼」的恐慌
4. **L3-L7 連續強 hook**,跨 lesson 連得緊,我願意一口氣玩到 L7
5. **vocab gate 機制 (tap-pairs)** 讓我每節都有「先撐起 4 個字再聽」的安全感

**不太好的地方**:
1. **listen-mc 大部分是 paraphrase 不是推理**,我感覺「答題像在驗證有沒有聽到 sentence」而不是「真的在用英文思考」 — 這是整章最大 risk,長期會被察覺
2. **"my dear" 第一次出現沒鋪墊**,我 0.3 秒卡住的時候很容易被打斷沉浸感
3. **L6 emoji-pick 🐪 直接給駝峰** — L6 應該是高潮反而給洩答題
4. **"Humph!" Web Speech 發音 [hʌmp]** 不像擬聲詞,Kipling 的擬聲幽默丟了 30%

**整體 verdict**:Ch4 是目前最強 narrative 章節 (Ch1-3 中 Ch3 醜小鴨次之)。**主結構穩、情緒設計成功、Kipling voice 大致保留**。但 **Q author 的 paraphrase 偷懶問題系統性出現**,P1-2 必須修才能匹配 narrative 品質。建議 ship 但**緊接 follow-up commit 修 P1-2 + P1-3 + P1-4** (9 題改寫,工作量 ~2hr)。
