Today's angle: **Memrise MemBot AI conversation pattern**
Source: https://www.memrise.com/app/en/conversation

## A. Angle research (5 bullet)

- **Turn-based chat scaffolding** — MemBot presents a scrollable chat thread where the AI character speaks in left-aligned bubbles; user responses appear right-aligned. The conversational frame ("you are talking to X") replaces the quiz-test frame ("you are being examined").
- **Persistent character identity** — MemBot's avatar (face + name) anchors every screen: avatar persists in a header chip throughout the session so the learner never forgets "who" is guiding them; builds emotional continuity across 10–15 turn sessions.
- **Speaker-differentiated bubble style** — distinct background colours for each speaker (AI = soft teal/blue; user = white/light). Past exchanges scroll above the active question, giving spatial context ("we were talking about X three turns ago").
- **Inline correction in chat stream** — when the user picks a wrong option, the correction appears as a follow-up bubble FROM the character ("Hmm, actually the word is…"), not as a separate feedback panel below the options. Correction feels like the character responding, not a system interrupt.
- **Conversation-phase signalling** — MemBot uses microcopy like "Your turn!" or a brief ellipsis pulse ("…") before its next message appears, creating a sense that the character is "thinking" — delays are presented as conversational timing, not loading latency.

## B. Pickup 現狀 (對齊這 angle)

- **SpeakerBadge exists but orphaned on archive** — `renderers.tsx:130-144` has a `SpeakerBadge` (mochi/grandma/hana/narrator pill) rendered on the active question, but `LessonPage`'s `NarrativeLine` (history display) ignores speaker entirely. Past lines look identical regardless of who spoke them.
- **History is `string[]` — speaker info permanently discarded** — `LessonPage.tsx:31` `history` is `string[]`; `onAdvance(snapshot?)` only takes a string. The `speaker` field from `RawQuestion` is never passed into the history array, making it impossible to attribute past lines.
- **NarrationRenderer hardcodes calico-anchor avatar** — `renderers.tsx:174` always renders `calico-anchor.webp` (Mochi cat) even when `q.speaker === 'grandma'` or `'hana'`. The SpeakerBadge pill correctly says "Grandma 👵" but the avatar shows a cat — speaker identity is contradicted.
- **No chat-bubble direction differentiation** — all history lines (NarrativeLine) are left-aligned flat rows with a speaker icon. MemBot pattern of left=character / right=player response is absent; Pickup history reads as a monologue even when multiple speakers appear.
- **Explanation panel is separate system UI, not in-character response** — `renderers.tsx:116-120` `<Explanation>` is a `borderLeft: '3px solid #c8a878'` sidebar card below the answer buttons. It reads as "the app is explaining", not "Grandma is responding to your answer". Breaks the conversational frame.
