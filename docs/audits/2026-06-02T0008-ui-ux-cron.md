## A. Angle research (5 bullets)

- **Mastery tiers per skill**: KA uses a 4-tier system — Attempted (gray) / Practiced (blue) / Mastered (dark blue) / Level Up challenge unlocked (crown). Each node on the course map carries a visual badge, not just ✓ vs locked. This gives learners granular "where am I exactly" feedback.
- **Unit summary bar**: Every unit card on the course page shows a horizontal fill bar ("30% mastered") computed from child-skill mastery states. Learners immediately see which chapter needs attention before diving in.
- **Energy Points / XP prominently shown**: KA shows running XP in the top HUD and on the profile page, always live-derived from localStorage/DB, never a hardcoded dash. Profile has a "Stats this week" card that breaks down daily XP.
- **Mastery Challenge as chapter gate**: After all skills in a unit are Practiced, a "Mastery Challenge" node unlocks — mixed questions from the entire unit. Passing promotes skills from Practiced → Mastered. This extends replay value significantly.
- **Course progress ring on dashboard**: Profile page shows a donut ring for each enrolled course — % lessons mastered. On mobile, these rings are 56px and sit in a horizontally-scrollable strip.

## B. Pickup 現狀 (對齊 KA angle)

- **Chapter cards (ChaptersPage)** are uniform orange buttons — no completion % bar, no "8/24 lessons done" annotation. All 8 chapters look identical regardless of progress.
- **MapPage nodes** have 3 states (locked/done ✓/next-pulse), but no mastery tier — a lesson passed by blind-retry after 5 tries looks identical to a first-try-correct lesson.
- **ProfilePage XP/Coins/Level all show "—"** despite `readXp()`, `readCoins()`, and `levelProgress()` being fully implemented in `src/data/xp.ts` and `src/data/coins.ts`. The imports were never wired up.
- **MapPage HUD coin** is hardcoded `value="0"` — `readCoins()` is never called here either.
- **TasksPage daily goals** are hardcoded `done: false` for all three tasks — no actual tracking of "completed 1 lesson today" or "accuracy ≥ 80%".
- **No chapter-level % summary**: `readCompletedLessons(chapter)` returns a `Set` that contains the count, but no page aggregates this into a visible "N/24 lessons" label on chapter selection cards.
