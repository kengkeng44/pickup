## A. Angle research — Khan Academy progress visualization patterns

- **Unit mastery heat-map**: each skill cell colored by mastery level — Gray (not started) → Red (attempted) → Yellow (practiced) → Blue (level 1 mastered) → Dark blue (level 2 mastered) → Green (perfect). Visual density communicates progress at a glance.
- **Course overview bar**: single %-fill bar at the top of every subject page aggregates mastery across all units; learner sees "47% of Algebra 1 mastered" without opening any unit.
- **"Review recommended" badge**: orange dot on skills whose SRS-decay has dropped below threshold — tells learner exactly *which* skills to revisit, not just "you should review".
- **Level-gated unlock + teaser**: locked units show greyed-out skill list; tapping a locked skill shows a mini-preview and the unlock requirement ("Complete Unit 3 to unlock"), removing fear-of-unknown friction.
- **Activity calendar heatmap**: weekly/monthly green-square calendar on learner profile; single number streak replaced by spatial pattern — learner sees "I practiced Mon/Wed/Fri last week" which motivates filling gaps.

## B. Pickup 現狀 (對齊 Khan Academy lens)

- **MapView.tsx**: `progress: 8, total: 24` **hardcoded** — never reads `readCompletedLessons(ch)`. Every player always sees 8/24 regardless of actual lesson completions. KA equivalent: course overview bar stuck permanently at 33%.
- **AlertsPage.tsx**: all 8 achievements `earned: false` hardcoded — no evaluation logic reads streak count, XP total, or completed lesson set. Dead display: a KA mastery heat-map where every cell is permanently gray.
- **TasksPage.tsx**: `streak` pulled from `useRunStore(s => s.streak)` (in-memory, resets on page reload) not `readStreak()` from persistent storage. Shows wrong value every cold start.
- **SRS review queue invisible**: `srsReviewBatch()` in `storyKitten.ts` correctly tracks wrong-answered questions but nothing in the React UI surfaces "N items due for review". KA's "review recommended" badge — the highest-ROI retention feature — is completely absent.
- **`levelProgress().fraction` orphaned**: `xp.ts` exports `levelProgress()` returning `{ fraction 0..1 }` but ProfilePage only shows raw XP number + "L{n}" text. No progress bar toward next level. GameHUD had this in legacy v1.9.37 (audit-2 F3) but the React ProfilePage regression went uncaught.
