Today's angle: **LingQ known-words counter (vocabulary mastery)**
Source: https://www.lingq.com/en/learn/en/web/#/learn/reader

## A. Angle research (5 bullet)

- **Known-words counter is LingQ's north-star metric.** The app shows "you know 7,432 words" prominently on home. This single number grows every session and anchors the learner's sense of real progress — distinct from streaks or XP (both ephemeral). Pickup has no equivalent vocabulary-growth surface.
- **Color-coded word status across all texts.** Every word in a reading is visually tagged: blue = new (first encounter), yellow = "lingq saved" (actively learning), white/blank = already known. The color density gives learners an at-a-glance sense of how hard a text is before they start it.
- **"Lingqs created" as daily goal metric.** LingQ's daily goal chip tracks how many new words the user saved today. The goal is binary (saved ≥ N lingqs → day counts) and tightly integrated with the streak system. Every word lookup is a persistent, counted event.
- **Vocabulary density preview on text list.** Before opening a text, LingQ shows a bar or % like "93% known words" — so advanced users avoid boring texts and beginners pick entry-level ones. Pickup shows no vocabulary difficulty signal per lesson.
- **Fire-and-forget lookups are UX waste.** Apps that show translations on tap but discard the event (no persistence) lose the most valuable data: which words the user needed help with. LingQ converts each lookup into a SRS card automatically. Pickup's WordHint tap is currently fire-and-forget.

## B. Pickup 現狀 (對齊這 angle)

- **MapPage HUD is hardcoded dead.** `const xp = 0; const coins = 0; const level = 1;` (MapPage.tsx lines 161–166) — all three values are hardcoded. The Crown level tier and progress bar in the HUD always show L1 / 0% regardless of actual play history. `readXp()`, `readCoins()`, `levelForXp()` all exist in `src/data/` but MapPage never imports them.
- **LessonPage CompletePanel awards fake XP.** CompletePanel computes `const xp = correct * 10` for display only (LessonPage.tsx line 149) but never calls `addXp()`. The persistent XP store (`src/data/xp.ts`) is therefore never updated from the new React lesson system — Crown level cannot advance through normal play.
- **ProfilePage Stats show literal dashes.** `<Stat label="XP" value="—" />` and `<Stat label="Coins" value="—" />` (ProfilePage.tsx lines 41–42) — `readXp()` and `readCoins()` are not imported. The stat grid exists purely as UI chrome with no live data.
- **TasksPage daily goals always false.** All three goals (`done: false`) are hardcoded constants — they never reflect actual lesson completions, accuracy, or narration listens. The daily motivation loop is entirely cosmetic.
- **No vocabulary mastery counter anywhere.** WordHint provides on-demand Chinese translations when a user taps a dashed-underline word, but the event is fire-and-forget. There is no "words looked up," "words encountered," or "vocabulary size" counter visible anywhere in the app.
