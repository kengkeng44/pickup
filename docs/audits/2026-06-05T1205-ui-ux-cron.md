## A. Angle research (5 bullets)

- **Chat-bubble UI model**: MemBot renders each turn as a speech bubble — bot left-anchored with avatar, user right-anchored. Visual left/right rhythm communicates "dialogue" at a glance; learner immediately understands whose turn it is.
- **Character-consistent avatar**: The MemBot avatar is always the same sprite across all sessions. Character identity doesn't change per-lesson. This builds parasocial attachment (Reeves & Nass "media equation" — learners respond to characters as social actors).
- **Scenario framing per session**: Every MemBot conversation opens with a 1-sentence context card ("You're ordering coffee in Madrid"). Sets expectation and gives the learner a role. Works equally for adults + children (role-play is core childhood learning).
- **Per-turn emotional reaction**: MemBot responds to correct answers with short affirmations in-context ("Great! The waiter understood you!"), not just a ✓ icon. Incorrect answers get character-voiced corrections ("Hmm, I'm not sure what you mean — try again?"). Feedback stays in the world of the story.
- **Progressive hint system**: If user is stuck MemBot offers a hint (dim the hardest distractor, or show first letter). Used sparingly — not a "show answer" button. Costs nothing but delays the conversation, creating mild tension.

## B. Pickup 現狀 (對齊 Memrise MemBot lens)

- **SpeakerBadge chip exists but isn't spatial**: `SpeakerBadge` in renderers.tsx renders a small pill (`mochi 🐱` / `grandma 👵`) above each entry — correct concept, but all entries stack identically left-aligned regardless of who's speaking. MemBot's key insight is that *position* communicates speaker identity faster than a label chip.
- **NarrationRenderer has no Continue button**: Spec R2 explicitly lists `"繼續 ↓ Continue" green button (.pickup-narration-continue)` for narration entries. Current code auto-advances via `onEnd` + 2000ms dwell — the Continue button is absent. Children 8-12 need agency; forced auto-advance breaks conversational pacing (you can't "read ahead" or re-read a grandma sentence).
- **Lesson history (NarrativeLine) strips speaker identity**: `NarrativeLine` renders past entries as identical plain text rows + replay icon. Once an entry scrolls into history, there's no visual cue that it was grandma vs. narrator vs. Mochi — the cast dissolves into an anonymous wall of text. MemBot keeps each past turn visually tagged to its speaker.
- **All speakers share the same calico-anchor.webp**: `NarrationRenderer` always shows `/mascots/calico-anchor.webp` (Mochi's photo) as the bubble avatar, even when `q.speaker === 'grandma'`. Grandma's lines appear as if spoken by the cat. Character confusion is the #1 narrative immersion killer.
- **EmojiPickRenderer reveal has no character anchor**: The `reveal` phase shows calico-anchor + text, then a CTA "聽 Mochi 的故事 →". But there's no grandma present, even though she is the storyteller in Ch1. The emotional payoff (grandma nods approvingly) is missing.

## D. Top 3 actionable (S effort)

### D1 — P1: NarrationRenderer Continue button (30 min)
**File**: `src/react-app/renderers.tsx`, `NarrationRenderer`  
**Change**: After TTS fires, show a green `繼續 ↓` button (`#7ac74a`, same style as other CTAs). Button onClick → `advanceOnce()`. Keep fallback timer as safety net but bump it to `Math.max(12000, ...)` so it doesn't race the button. Aligns with spec R2 `.pickup-narration-continue`.  
**Why now**: Every lesson starts with narration. Missing Continue = auto-advance silently skips content children didn't finish reading → comprehension gap before Q1.

### D2 — P1: Speaker-to-avatar mapping (20 min)
**File**: `src/react-app/renderers.tsx`, `NarrationRenderer` avatar `<img>` line  
**Change**: `const SPEAKER_AVATAR: Record<string, string> = { mochi: '/mascots/calico-anchor.webp', grandma: '/mascots/iso-grandma.webp', hana: '/mascots/iso-shiba.webp', narrator: '/mascots/iso-grandma.webp' }` — replace hardcoded `calico-anchor.webp` with `SPEAKER_AVATAR[q.speaker ?? 'narrator']`.  
**Why now**: Grandma is the storytelling protagonist of v2.0. Using Mochi's photo for grandma's lines is a brand mismatch. Iso-grandma asset already exists at `/mascots/iso-grandma.webp`.

### D3 — P2: storyBeat sub-header in lesson (10 min)
**File**: `src/react-app/pages/LessonPage.tsx`, lesson header `<div>` (after q-counter chip)  
**Change**: `{lesson.storyBeat && <span style={{ fontSize: 12, color: '#8b6f4a', fontWeight: 700, marginLeft: 8, fontStyle: 'italic' }}>{lesson.storyBeat}</span>}` — one line.  
**Why now**: Zero data cost (field already in JSON), gives returning players instant context, directly mirrors MemBot scenario card pattern. Especially helpful for Ch1 L5+ where lesson content is mid-story.
