# Pickup UI/UX Cron — 2026-06-04 09:08 UTC

Today's angle: **Drops 5-min session pacing**
Source: https://languagedrops.com/ (app UX pattern, widely documented in AppStore reviews + edu-app teardowns)

---

## A. Drops pattern (5 bullets)

1. **Hard 5-min timer with circular ring** — Prominent countdown arc in session header. User sees time draining, not question count. Creates urgency without "X of Y" anxiety; ring fills/drains viscerally.
2. **Pre-session topic card with word count** — Each topic card shows "23 words · ~5 min" before you tap. No unknown commitment — player chooses knowing the cost.
3. **"Done for today" gate after session ends** — Free users hit a friendly wall: "Great work! Come back tomorrow." Explicit stop signal. Pro = unlimited; this binary is the key monetization mechanic AND UX benefit: session has a clear end.
4. **Sub-2s auto-advance per correct** — Each tap→correct takes ~1s before next item appears. 5 min × ~1s cadence = ~300 words possible. Drops optimises for velocity, not depth.
5. **Post-session "words learned today" counter** — Session-end screen shows only today's new words + cumulative total (e.g. "5 today / 420 total"). Small delta against large total = mastery momentum.

---

## B. Pickup 現狀 (對齊 Drops lens)

1. **Node has no time estimate** — 24 map nodes look identical. Tapping a node commits you to 5–14 questions with zero time signal. `q1/14` chip appears only after entry. Player can't plan a "quick 3-min break".
2. **No "done for today" signal** — CompletePanel says "Lesson complete!" then navigates back to map with 23 remaining nodes visible. No "today's target met" moment → sessions feel open-ended, counter to "下班逃逸" positioning.
3. **Auto-advance 3–5s is 3× slower than Drops cadence** — `ADVANCE_CORRECT_MS = 3000` (listen-tf/mc) + narration Continue button. Drops ~1s. A 14Q lesson at 3s/correct ≈ 42s of waiting. Idle time erodes "flow" state.
4. **q-counter chip "q3/14" is low-urgency text** — A tiny amber pill in the corner. Drops' circular ring is visually central and kinetic. Pickup's chip neither conveys time nor creates pacing tension.
5. **TasksPage daily goal "正確率≥80%" is a stub** — Hardcoded `done: false`. Two of three daily tasks can't fully resolve (accuracy is fake, streak is real). Drops shows real metrics immediately after session.

## D. Top 3 actionable (S effort)

### D1 — Node 時間預估 micro-label (MapPage)
**檔案**: `src/react-app/pages/MapPage.tsx`, node button render loop (~line 270)
**改法**: 在 `<button>` 內 icon 下加：
```tsx
<span style={{ fontSize: 10, color: '#8b6f4a', fontWeight: 700, marginTop: 2 }}>
  ~{Math.ceil(l.questions.length * 15 / 60)}min
</span>
```
`l.questions.length` 已在 Lesson interface 的 `questions: unknown[]` 裡，無需 fetch。
**效果**: 玩家在 map 上就能說「我現在只有 3 分鐘，選這個節點」—— "下班逃逸" 品牌承諾落地。

### D2 — CompletePanel "今日累計" 4th stat card
**檔案**: `src/react-app/pages/LessonPage.tsx`, `CompletePanel` component (~line 130)
**改法**: `correct` / `total` 計算後：
```tsx
let todayLessons = 0;
for (let ch = 1; ch <= 8; ch++) todayLessons += readCompletedLessons(ch).size;
```
加第 4 張 `<Stat label="TODAY" value={`${todayLessons}課`} color="#d68a52" bg="#fff7e8" />`。
把 `display: 'flex'` 的 gap 改 8px,4 卡各 `flex:1`。
**效果**: Drops 式「今天做了 N」mastery momentum signal; 無需新 store slice。

### D3 — LessonPage header 進度 bar
**檔案**: `src/react-app/pages/LessonPage.tsx`, header div (~line 80)
**改法**: q-counter span 右側：
```tsx
<span style={{ display:'inline-block', width:72, height:4, background:'rgba(122,104,80,0.15)', borderRadius:2, marginLeft:8, verticalAlign:'middle' }}>
  <span style={{ display:'block', width:`${Math.round((idx/lesson.questions.length)*100)}%`, height:'100%', background:'#e7a44a', borderRadius:2 }} />
</span>
```
**效果**: 視覺進度 bar 取代純數字 chip 的抽象感; idx=0 時空 bar 激勵「往前走」; 完成時 bar 填滿即時回饋。
