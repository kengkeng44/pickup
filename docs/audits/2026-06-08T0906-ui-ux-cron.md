## A. Angle research (5 bullet)

- **Front/back reveal + self-rating**: Anki's core loop is "see front → attempt recall → flip → rate: Again / Hard / Good / Easy." The rating directly sets the next review interval (e.g. Again=1min, Good=1day, Easy=4days). Self-rating gives users agency and dramatically increases long-term retention vs binary correct/wrong.
- **Queue counter on home screen**: Anki's deck list always shows "New · Learn · Due" counts prominently (e.g. "0 · 3 · 42"). This creates pull: users see the obligation/opportunity before they commit to a session.
- **Visual card-type tagging**: During a session, Anki colour-codes card backs — blue dot = new, green = review, red = relearn. Users can always see what kind of card they're on.
- **Interval transparency**: After rating, the next intervals are shown ("again in 1 min · good in 1 day"). This demystifies the algorithm and builds trust. AnkiDroid shows it above the rating buttons in real-time.
- **Lapse tracking + "leech" concept**: Cards failed many times become "leeches" and are suspended. Anki makes this visible (leech tag + count in stats). The system communicates that chronic failures warrant a different strategy, not just more retries.

## B. Pickup 現狀 (對齊這 angle)

- **SRS queue completely invisible**: `readSrsQueue()` exists in `storyKitten.ts:362` but the queue size is never displayed to the user anywhere — not on MapPage, not on TasksPage, not in any HUD. User has zero visibility into their backlog.
- **v2.0 LessonPage never writes to SRS**: `onAnswer()` in `LessonPage.tsx:~109` pushes to `answerLog.current` and fires PostHog but **never calls `addToSrs(q.id)`**. Every wrong answer in v2.0 lessons is silently lost from SRS. (Confirmed: `grep -n "addToSrs" src/react-app/pages/LessonPage.tsx` → zero results.)
- **Binary SRS fate, no scheduling**: Wrong answer = added to SRS array. Correct answer on retry = removed immediately. There is no interval, no ease factor, no "how hard was this." The SRS is effectively a simple "missed questions list" with no spaced-repetition algorithm behind it.
- **Reviews silently injected (legacy only)**: `runStore.ts:328` injects up to 3 SRS cards at the start of a story chapter (legacy PlayScene). No visual differentiation between review vs new cards. User doesn't know they're doing a review.
- **No standalone review entry point**: There's no way to access the SRS review queue independently of story progression. CardsPage exists but shows character collectibles, not missed vocabulary.

## D. Top 3 actionable (S effort)

### #1 — LessonPage `onAnswer` 補 `addToSrs()` 呼叫 (P0, 15 min)

**Why**: v2.0 的整個 SRS 系統在 LessonPage 根本無效。每堂課的答錯題都進黑洞，不進複習庫。所有後續依賴 SRS queue（notification gates、legacy review inject）全都餵到錯誤的舊 PlayScene 資料。

**Where**: `src/react-app/pages/LessonPage.tsx`

**Fix**:
```ts
// top import
import { addToSrs } from '../../data/storyKitten';

// in onAnswer callback (~line 112)
const onAnswer = (userIdx: number, isCorrect: boolean) => {
  answerLog.current.push({ q, userIdx, isCorrect });
  if (!isCorrect) { try { addToSrs(q.id); } catch {} }  // ← add this
  // ... existing PostHog track call
};
```

---

### #2 — CompletePanel「N 題加入複習」反饋標 (P1, 20 min)

**Why**: Anki 每次 session 結束都明確顯示「N cards learned/reviewed」。Pickup 的 CompletePanel 顯示 XP/Accuracy/Time 但完全不提 SRS。讓用戶知道錯誤有被記住，強化「錯了沒關係、會補救」的安全感（兒童受眾特別重要）。

**Where**: `src/react-app/pages/LessonPage.tsx` `CompletePanel` function, after the 3-Stat row (~line 305)

**Fix**:
```tsx
{log.filter(a => !a.isCorrect).length > 0 && (
  <div style={{ fontSize: 12, color: '#8b6f4a', fontWeight: 700, marginBottom: 12 }}>
    📚 {log.filter(a => !a.isCorrect).length} 題加入複習庫 · Added to review
  </div>
)}
```

---

### #3 — 答錯揭曉後加「加入複習庫」microcopy (P2, 15 min)

**Why**: Anki 的核心心理是「答錯 = 下次還會見到」— 錯誤有 consequence 但也有 closure。Pickup 目前答錯只有紅色反饋，沒有 forward-looking 說明。在 `<Explanation>` 下加一行「📚 加入複習庫」給用戶確定感：這道題沒過，但系統記住了。

**Where**: `src/react-app/renderers.tsx` `Explanation` component or inline after reveal in ClozeRenderer/ListenMcRenderer

**Fix**: In `Explanation` component, add optional `showSrsHint` prop; OR simply append below `<Explanation>` where `!isCorrect`:
```tsx
{revealed && !isCorrect && (
  <div style={{ fontSize: 11, color: '#a89c80', marginTop: 4, fontWeight: 700 }}>
    📚 加入複習庫 · Added to review
  </div>
)}
```
