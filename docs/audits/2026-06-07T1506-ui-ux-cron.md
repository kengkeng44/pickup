## A. Angle research — Duolingo Streak/League patterns

1. **Layered unlock pacing**: Duolingo introduces streak motivation only at D7+ and league at D14+. New users (D0-D6) see simplified HUD with zero competition pressure — no league ranking, no streak count shown in achievement context until it "feels worth protecting."
2. **Streak Freeze visual hierarchy**: Freeze is a dedicated shop item (cost: 10 gems), with explicit "equipped/not equipped" states. The equip confirmation screen shows the freeze "activating" — users feel agency, not safety-net passivity. Duolingo never shows freeze as a rescue; it's always pre-purchase.
3. **Weekly League rhythm**: Users are sorted into 30-person cohorts, reset every Monday. Diamond tier introduces "Demotion Zone" (bottom 5 drop) and "Promotion Zone" (top 3 advance) — asymmetric stakes create loss-aversion pull stronger than pure reward. Tier badges appear on public profile.
4. **Streak milestones as story beats**: Every 5 → 7 → 10 → 30 → 100 → 365 days triggers a separate full-screen celebration overlay (not just a toast), with streak-specific copy ("You've been learning for 30 days! That's like reading 6 books!"). The number itself becomes the narrative.
5. **XP Goals as daily commitment device**: Duolingo lets users choose a daily XP goal (Casual 10, Regular 20, Serious 30, Intense 50) during onboarding. The HUD progress ring fills toward *their* self-declared target, not a universal bar — personalizing the success condition reduces "not worth trying today" dropout.

## B. Pickup 現狀 — 對齊 Streak/League lens

1. **Streak core: solid**. `src/data/streak.ts` has freeze auto-consume, warm microcopy framing ("Mochi 幫你保住"), `readStreak()` / `readFreezes()` wired into MapPage HUD and TasksPage hero card. Foundation is Duolingo-parity.
2. **Freeze UX: reactive, not proactive**. Freeze is a HUD number (🧊 2) that auto-fires silently. Duolingo's freeze requires explicit equip + confirmation — the *ritual* of equipping creates attachment. Pickup's freeze just happens, so users may not notice it was "used."
3. **Streak milestones: missing celebration**. 3-day and 7-day badges exist in `AlertsPage.tsx` (`earned: false` hardcoded — they never actually unlock). No full-screen streak milestone overlay at 7 / 30 / 100. A user at 10 days sees the same HUD flame as day 1.
4. **No weekly goal / XP goal selector**: Duolingo's onboarding captures "how much time per day?" — producing a personalized progress ring. Pickup shows a flat XP count with no self-declared goal context. The crown level progress bar (v1.9.37) fills toward arbitrary level boundary, not a user-chosen daily target.
5. **No league / social layer**: Pickup is explicitly single-player (per CLAUDE.md "no friend graph" decision, replaced by mascot relatedness). This is intentional. However, *leaderboards within family accounts* (parent + child) or class-level competition are a natural child ELT fit that Duolingo schools exploits heavily. Currently zero social hooks.

## D. Top 3 actionable (S effort, implement now)

### D1 — Wire achievements `earned` to real data (S, 1h)
`AlertsPage.tsx` hardcodes `earned: false` for all 8 badges. Fix:
```tsx
import { readStreak } from '../../data/streak';
import { readXp } from '../../data/xp';
import { readCompletedLessons } from '../../store/runStore';
// derive inside component:
const streak = readStreak();
const xp = readXp();
const ch1done = readCompletedLessons(1).size;
const earned = {
  1: ch1done >= 1,           // 第一步
  2: streak >= 3,            // 三日連擊
  3: streak >= 7,            // 七日狂熱
  4: false,                  // 完美主義 — needs session accuracy data, defer
  5: readCompletedLessons(2).size > 0, // 探索者
  6: ch1done >= 24,          // 說書人 (Ch1 = 24 lessons)
  7: xp >= 1000,             // 神射手
  8: false,                  // 畢業生 — needs Ch8 complete, defer
};
```
Achieves: badges actually earn → credibility + "aha" when user comes back after 3 days.
