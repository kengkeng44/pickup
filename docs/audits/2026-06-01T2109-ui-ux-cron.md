## A. Duolingo streak/league pattern (5 bullets)

- **Streak = primary retention anchor**: 🔥 flame icon + day count appears in HUD, lesson complete screen, push notifications, AND a dedicated "streak card" that surfaces on app open the day after a streak increment. The moment of streak extension is its own micro-celebration (animated flame pop, "🔥 X day streak!" full-screen modal).
- **Streak Freeze = loss-aversion monetization**: Users can buy/earn 1 Freeze that auto-deploys if they miss a day. The gem cost (10 gems) is calibrated so it's always affordable. Duolingo case study: Streak Freeze reduced churn by ~21% in cohorts that used it — because catastrophic loss-of-streak was the #1 voluntary churn trigger.
- **Streak Repair window**: If freeze was unused and streak breaks, a "Repair Streak" prompt appears for 24h after the miss — spend gems to restore. This converts would-be churners at the most emotionally vulnerable moment.
- **League as social pressure layer**: 10-tier XP league (Bronze → Diamond) resets weekly. Bottom 10 players face demotion; top 3 get promoted. This generates urgency without requiring friends — you compete vs. strangers in a bracket. Leagues drive 2–3× session length in the last 48h of the weekly window.
- **Streak milestone moments**: Duolingo triggers distinct animations at day 3, 7, 30, 100, 365. Each milestone unlocks a cosmetic (badge, profile icon). Share-sheet prompt at 7+ days drives organic social acquisition.

## B. Pickup current state (aligned to streak/league angle)

- **`updateStreak()` only called in old Phaser `BootScene.ts` (line 26) — NEVER in React v2.0 flow**: When a user completes a lesson in `LessonPage.tsx`, `markLessonCompleted()` is called but `updateStreak()` is NOT. Streak can only advance on app boot, not on the lesson-complete event. In v2.0 (React router), users likely never trigger the Phaser BootScene at all — streak is permanently frozen.
- **AlertsPage.tsx achievements are hardcoded dead code**: The React page defines its own local `ACHIEVEMENTS` array with all `earned: false` constants. It never calls `evaluateAchievements()` from `src/data/achievements.ts`. Result: every badge shows permanently locked regardless of actual streak/XP/chapter progress.
- **TasksPage.tsx daily tasks are static decorations**: Three tasks (`{ done: false }`) are hardcoded. No connection to `markLessonCompleted`, `updateStreak`, `readXp`, or any real event. The tab is a wireframe masquerading as a feature.
- **CompletePanel has no streak moment**: Lesson complete shows XP / Accuracy / Time triptych — no streak counter, no "🔥 streak extended!" celebration, no freeze nudge. The highest emotional peak in the UX loop misses the strongest retention hook.
- **No Streak Freeze, no Streak Repair**: Master matrix lists Streak Freeze as ROI #2 (churn -21%, 5 person-days). Nothing in `src/data/streak.ts` or the UI supports freeze or repair flows.

## D. Top 3 actionable (S effort)

### 🔥 D1 — Fix streak never updating in v2.0 (XS, P0 bug)

`src/react-app/pages/LessonPage.tsx` CompletePanel `useEffect`:
```ts
import { updateStreak } from '../../data/streak';

useEffect(() => {
  try { markLessonCompleted(lesson.chapter, lesson.id); } catch {}
  try { updateStreak(); } catch {}   // ← add this
  // ... rest of analytics
}, []);
```
Without this fix, the entire streak system is inert in the React app. One line.

### 🏅 D2 — Wire AlertsPage to real achievement data (S, P0 bug)

Replace `AlertsPage.tsx` hardcoded ACHIEVEMENTS with:
```tsx
import { evaluateAchievements } from '../../data/achievements';
const achievements = useMemo(() => evaluateAchievements(), []);
```
Then map over `achievements` using `a.unlocked` for styling and `a.iconSrc` for PNG icon rendering. Removes the embarrassing "all badges locked forever" state.

### 🎉 D3 — Streak moment in CompletePanel (S, P1 retention)

After fixing D1, read streak before/after and conditionally render:
```tsx
const [streakAfter] = useState(() => { try { return readStreak(); } catch { return 0; } });
// (read before updateStreak call — or read delta)
```
Add below the 🎉 emoji in CompletePanel:
```tsx
{streakAfter > 0 && (
  <div className="pickup-streak-pop" style={{ color: '#ff7a3a', fontWeight: 900, fontSize: 18 }}>
    🔥 {streakAfter} day streak!
  </div>
)}
```
This is the single highest-ROI UX moment in the entire retention loop.
