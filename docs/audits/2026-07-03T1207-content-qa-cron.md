### B3 — Narrative Voice / Pacing Improvements (Zero-violation proposals)

Even with no hard-rule violations, three patterns reduce ELT effectiveness for 8-12 children:

1. **Compound-stimulus position density (A2 primacy/recency)**: 18/36 Ch17 MC stimuli are compound sentences ("She cooked the meals. She cleaned the house."). The tested information consistently appears in the **first clause** of compound stimuli — creating primacy-advantage bias (first info recalled best). Mix: ~⅓ test first clause, ⅓ test second clause, ⅓ test cross-clause integration.

2. **WH-question stem redundancy**: Ch18-20 reuse "What did X do?" / "How did X feel?" as the question stem for ≥5 questions per chapter. Per R7, WH distribution should be varied: add more "Why", "Which one", "When" prompts to increase sub-skill variety.

3. **comprehension explanationZh register**: In Ch22, explanations like "這告訴我們一個道理：持之以恆的學習才能成就偉業。" use adult Confucian register. For 8-12 age group, rephrase as "孟媽媽搬了三次家，就是為了讓孟子在好環境學習——真的很用心！" (story-first, not moral-first).

## 🔬 Architecture Recommendation (對齊業界 2026)

**ARCH-REC #109: X65_CORRECTINDEX_POSITION_MONOTONE_LINT**

### Background (業界掃描)

2025 simulation study (T&F, *Applied Measurement in Education* Vol 38): confirmed that **pure randomization** of answer key positions is the only guideline not linked to validity threats. Balancing (ensuring 25% each) is second-best. **Neither** was found in Ch3/4/17/18/32/33 — instead, systematic LLM-generation artifacts created monotone and 2-cycle patterns.

Research from "Guess Where" (psychometric literature) quantified: correct answers in predictable positions reduce item discriminability and allow test-wise examinees to achieve 50–100% without content engagement.

For Pickup's 8-12 child audience, position bias is especially harmful: children are fast pattern-detectors. A motivated child reaching Ch32 discovers in under 30 seconds that "A is always right" — turning a listening exercise into a game of button position, not English comprehension.

### Verdict: ✅ Directly applicable to Pickup architecture

Pickup uses static JSON files (`lessons-ch*.json`) — easy to lint at build time.
`validate-lessons.js` already parses the schema; extending it with a position-distribution check is a < 50-line addition.

### Proposed implementation

Extend `tools/validate-lessons.js` with a `checkPositionDistribution(lessons, chN)` function:

```js
function checkPositionDistribution(lessons, chN) {
  const seq = lessons
    .flatMap(l => l.questions || [])
    .filter(q => ['listen-mc','comprehension'].includes(q.type))
    .map(q => q.correctIndex)
    .filter(ci => ci >= 0 && ci <= 3);

  if (seq.length < 6) return; // too few to lint

  const dist = [0,1,2,3].map(i => seq.filter(x => x === i).length);
  const n = seq.length;

  // Rule 1: monotone — one index ≥ 90%
  const maxPct = Math.max(...dist) / n;
  if (maxPct >= 0.90)
    issues.push(`Ch${chN}: X65_POSITION_MONOTONE (idx_dist=${dist.join('/')}, n=${n})`);

  // Rule 2: perfect 2-cycle
  const isPerfectAlt = seq.length >= 6 &&
    seq.every((v, i) => v === (i%2===0 ? seq[0] : seq[1])) &&
    seq[0] !== seq[1];
  if (isPerfectAlt)
    issues.push(`Ch${chN}: X65_POSITION_ALTERNATING ([${seq[0]},${seq[1]}] cycle, n=${n})`);

  // Rule 3: any position never appears (≥8 questions)
  if (n >= 8) {
    [0,1,2,3].forEach(i => {
      if (dist[i] === 0)
        issues.push(`Ch${chN}: X65_POSITION_STARVED (idx ${i} never correct, n=${n})`);
    });
  }
}
```
