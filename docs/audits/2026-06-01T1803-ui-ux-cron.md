## A. Drops session-pacing pattern (5 bullets)

- **Time promise upfront**: Drops' entire brand is "5 minutes a day." Before a session the timer ring is visible at entry — user knows *exactly* what they're committing to. This drives conversion from "browse" → "tap to start" because the ask is bounded.
- **Circular progress ring as primary session UI**: A shrinking ring around the central button replaces traditional progress bars. Visual + kinesthetic = user feels time flowing, not just counting questions.
- **Variety in rapid succession**: Drops rotates 4–6 mini-exercise types (swipe, tap-the-word, match, fill) within a single session. No two consecutive cards feel identical. Prevents the "nth of the same" fatigue plateau.
- **Hard gate after session = tomorrow is the ritual**: Once 5 min is up, the gate closes and you wait until next session. This is distinct from Pickup's "complete the lesson" model — but the key UX insight is *framing*: Drops gives users permission to stop with no guilt.
- **Instant global visual reward on correct answer**: The whole card background pulses green/animates before advancing. It's not just a button state change — the *entire context* celebrates, then instantly clears for the next item. Zero friction between dopamine hit and next card.

## B. Pickup current state (aligned to this angle)

- **No session duration signal**: MapPage nodes show identical paw icons with zero time/question-count preview. User cannot make a "can I do this now?" decision before tapping. `lessons-ch1.json` has `questions.length` per lesson (range 5–15), but nothing surfaces this to the map or node button.
- **Daily goals are UI theater**: `TasksPage` renders 3 hardcoded goals — "完成 1 個 lesson", "正確率 ≥ 80%", "聽完整個 narration" — all with `done: false`. `readCompletedLessons()` exists in `runStore.ts`; completing a lesson writes to localStorage; but `TasksPage` never reads it. Goals never tick ✓.
- **Coin HUD hardcoded zero**: `MapPage:52` passes `value="0"` to the coin `HudIcon`. `readCoins()` in `src/data/coins.ts` returns the real persistent count. User earns coins on correct answers (runStore:439 `addCoins(1)`) but the HUD always shows 0.
- **ProfilePage stats dead-end**: `ProfilePage` stat cards show `"—"` for XP and Coins (lines 40–41). `readXp()` and `readCoins()` are importable one-liners; the connection was never wired.
- **NarrationRenderer: no manual Continue CTA**: Spec R2 mandates a "繼續 ↓ Continue" green button (`pickup-narration-continue` class). Current `NarrationRenderer` (renderers.tsx:100-129) auto-advances on TTS end with a fallback timer — no button visible. Short sentences (≤5 words, TTS fires in ~1.5s) advance before the user processes the text.
