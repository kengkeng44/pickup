# Pickup UI/UX Cron — 2026-06-06 22:16 UTC

Today's angle: **Lingq known-words counter (vocabulary mastery)**
Source: https://www.lingq.com/en/learn/

## A. Angle research (5 bullet)

- **The big number is the product.** LingQ's central metric is a running total of "Known Words" shown in the top nav on every page. Users chase it obsessively — "I know 8,342 French words" — because it's concrete, transferable proof of vocabulary progress independent of lesson completion.
- **Word status ladder: New → Learning (1-3) → Learned → Known.** Every word encountered in reading/listening gets a status. Blue = brand new, yellow shades = in progress, white = known. Users explicitly "mark as known" or the system auto-promotes after N encounters.
- **Encounter counts fuel the counter.** A word seen for the first time in a story becomes "New" (status 1). Re-encountering it in subsequent lessons increments the counter. Marking it Known grants a coin reward. The vocabulary list is the SRS deck.
- **Vocabulary milestone rewards.** LingQ has explicit milestones at 100 / 500 / 1,000 / 2,000 / 5,000 Known Words. Each unlocks a badge. These milestones give learners a medium-term goal longer than "finish this lesson" but shorter than "finish a chapter."
- **Word-level data surfaces everywhere.** Profile page shows total known words by language. Individual lesson shows how many "new words" are in it (pre-lesson preview). Post-lesson stat screen shows "+N words added to vocabulary." The number is unavoidable — it's the core loop hook.

## B. Pickup 現狀 (對齊這 angle)

- **Zero word-level tracking.** Pickup tracks XP / Coins / Streak / Lesson progress / Cards collected — but has no "words encountered" or "words learned" counter anywhere. The word-hints.json has 771 entries but they are purely a lookup dict with no per-user encounter state.
- **WordHint taps are a goldmine being wasted.** `wireSentenceHints()` in `src/ui/WordHint.ts` fires a `pointerdown` handler on every `.word` span — the user tapping a word is the strongest possible "I don't know this word" signal. Currently: tooltip appears → dismissed → event vanishes. Nothing persists.
- **Profile stats are gamification metrics only.** `ProfilePage.tsx` shows Streak 🔥 / XP / Coins / Crown Level — none of these are vocabulary metrics. A learner finishing Ch1 cannot answer "how many English words do I now know?" from looking at the app.
- **CardsPage has a collection counter (unlockedCount/total) but it counts story characters, not vocabulary.** The framing is close — "N / total 張卡片" — but cards ≠ words. Vocabulary mastery has no analog surface.
- **Achievements in AlertsPage are streak/XP/chapter-based; zero vocabulary milestones.** "認識 50 個詞" or "第一個查單字" achievements don't exist. The 771-word dict implies a vocabulary syllabus nobody can see progress against.
