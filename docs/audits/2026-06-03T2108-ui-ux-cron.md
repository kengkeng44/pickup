## A. Babbel SRS pattern (5 bullets)

- **Review counter always visible**: home tab surfaces "N words due for review" as a persistent badge; user always knows how much review debt is waiting.
- **Interval ladder displayed**: Babbel shows the actual schedule — correct once → tomorrow, again → +4d, → +7d, → +14d, → +60d, → +6mo. User sees progress *along the forgetting curve*, not just a pass/fail.
- **Dedicated "Today" tab** merges new lessons + overdue reviews into one action list; review is a first-class daily task, not buried.
- **Multiple review modalities**: flashcard / listening / speaking / writing — same vocabulary recycled across 4 input modes, each deepening a different skill.
- **Dashboard stats**: "Words learned", retention %, study time are the *primary* profile metrics. Vocabulary count is the headline number, not XP or coins.

## B. Pickup 現狀 (對齊 Babbel SRS lens, 5 bullets)

- **SRS queue is invisible**: `wordwar.srs.kitten` key exists in localStorage (array of question IDs), but zero surface in any UI tab — user has no idea if they have 0 or 40 words waiting.
- **P0 silent bug — LessonPage.tsx never writes SRS**: `src/react-app/pages/LessonPage.tsx` (200 lines) has **zero** references to `addToSrs` / `removeFromSrs`. Wrong answers in the new v2.0 React flow never enter the review queue; the queue only grows from the deprecated Phaser `PlayScene`. Every lesson played since v2.0 ship = no SRS data accumulating.
- **No timestamps = no interval scheduling**: `addToSrs()` stores only the question ID string, no `addedAt` or `correctStreak`. All overdue detection is impossible; no "next review in 3 days" is achievable without a schema migration.
- **Achievements hardcoded `earned: false`**: `src/react-app/pages/AlertsPage.tsx` line 1–14 — all 8 badges always show as unearned, including basic ones ("完成第一個 lesson", "連續 3 天") that are trivially checkable from existing data layers.
- **Profile stats don't count vocabulary**: Profile shows XP / Coins / Streak / Crown Level — none of which signal "vocabulary mastered". No "詞彙庫" or "已學 N 個單字" — the primary progress metric users care about (Babbel, LingQ both lead with this).

## D. Top 3 actionable (S effort)

### 1. P0 — Wire SRS hooks in `LessonPage.tsx` CompletePanel (~15 min)

`src/react-app/pages/LessonPage.tsx` CompletePanel `useEffect`:

```tsx
// 在 markLessonCompleted 之後加:
import { addToSrs, removeFromSrs } from '../../data/storyKitten';
// inside useEffect:
for (const entry of log) {
  if (entry.isCorrect) removeFromSrs(entry.q.id);
  else addToSrs(entry.q.id);
}
```

This is the most impactful fix: every v2.0 lesson played since launch has contributed zero SRS data. One useEffect import + loop closes a 2-month silent regression.

### 2. P1 — TasksPage "待復習" card (~20 min)

Read `readSrsQueue` and show pending count as a 4th task card with amber "!" indicator when count > 0. Babbel's #1 daily engagement driver is surfacing the review queue count on the home / tasks screen.

### 3. P1 — Wire 4 basic Achievements in AlertsPage (~15 min)

`AlertsPage` currently renders a static array. Replace `earned: false` with real checks:
- badge 1: `readCompletedLessons(1).size > 0`
- badge 2: `streak >= 3`
- badge 3: `streak >= 7`
- badge 7: `readXp() >= 1000`

Achievements showing as always-unearned actively *damages* motivation — user sees a tab full of grey locked icons even after completing chapters.
