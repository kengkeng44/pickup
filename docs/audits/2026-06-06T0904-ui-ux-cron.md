## A. Angle research (5 bullet)

- **5-tier mastery per skill** — Not Started (grey) → Attempted (light) → Familiar (medium blue) → Proficient (dark blue) → Mastered (purple). Each exercise tile shows its tier with color + label. User always knows how well they know a thing, not just done/not-done.
- **Unit progress % display** — each unit card shows "X% Mastered" numeric readout + stacked color bar (breakdown of how many skills in each tier). One glance → course health.
- **Learner Queue / "What to focus on next"** — 2024 redesign added algorithmic next-action surface on the learner dashboard. System tells you "work on THIS" rather than letting you hunt for the next node.
- **Parent/teacher progress reports** — time spent, mastery per skill, assignment completion. Even in kids app, parents get "Progress reports" with visual charts per activity type.
- **Gems + weekly streaks on top of mastery** — gamification is additive, not the primary signal. Mastery % is the real progress currency; gems/streak are engagement nudges layered on top.

---

## B. Pickup 現狀 (對齊 Khan Academy angle)

- **Binary done/not-done** — map nodes are olive (done) or brown (current) or grey (locked). No granularity: a node completed on 1st try vs 3rd retry looks identical. Zero mastery tiers.
- **ChaptersPage: no per-chapter progress %** — 8 chapter buttons show title only. User cannot see "Ch1: 17/24 done" at a glance. Completely dark on overall journey state.
- **Achievements tab: hardcoded `earned: false`** — ALL 8 badges are permanently unearned. No logic reads localStorage XP / streak / completedLessons. The achievement system is fully inert.
- **Profile: raw numbers, no context** — Streak / XP / Coins / Crown Level shown as 4 stat boxes. No "you need 33 more XP for L4" bridging label. KA's equivalent tile shows progress toward next tier explicitly.
- **Tasks: "正確率 ≥ 80%" permanently ○** — the third daily task has a placeholder comment `(待 session log)` and always shows incomplete. A broken KPI creates learned helplessness; 8-12 kids notice a metric they can never clear.

## D. Top 3 actionable (S effort)

### 1. Wire Achievements to real data (`AlertsPage.tsx`)
Current state: all 8 achievements hardcoded `earned: false` — zero ever unlock regardless of user progress. This is the biggest trust-erosion issue for returning users who've been playing for days.

```tsx
// AlertsPage.tsx — replace static ACHIEVEMENTS const with a computed hook:
const streak = useRunStore(s => s.streak);
const xp = readXp();
let completedCh1 = 0, totalLessons = 0;
for (let i = 1; i <= 8; i++) {
  const s = readCompletedLessons(i).size;
  totalLessons += s;
  if (i === 1) completedCh1 = s;
}
const ACHIEVEMENTS = [
  { id: 1, icon: '🐾', name: '第一步',  desc: '完成第一個 lesson', earned: totalLessons >= 1 },
  { id: 2, icon: '🔥', name: '三日連擊', desc: '連續 3 天學習',     earned: streak >= 3 },
  { id: 3, icon: '⚡', name: '七日狂熱', desc: '連續 7 天學習',     earned: streak >= 7 },
  { id: 4, icon: '⭐', name: '完美主義', desc: '完成 5 個 lesson', earned: totalLessons >= 5 },
  { id: 5, icon: '👣', name: '探索者',  desc: '解鎖 Ch2',         earned: readCompletedLessons(2).size > 0 },
  { id: 6, icon: '📚', name: '說書人',  desc: '完成 Ch1 全章',    earned: completedCh1 >= 24 },
  { id: 7, icon: '🎯', name: '神射手',  desc: 'XP 達 1000',       earned: xp >= 1000 },
  { id: 8, icon: '🏆', name: '畢業生',  desc: '完成所有章節',     earned: totalLessons >= 192 },
];
```
Note: #4 original desc was "滿分完成 1 個 lesson" which requires per-session accuracy tracking not yet implemented — replaced with "5 lessons" as trackable proxy.

### 2. Add per-chapter progress % to ChaptersPage (`ChaptersPage.tsx`)
Each chapter button shows title only. 3 lines of JSX adds a progress pill — visually tells user "where am I across all chapters."

```tsx
// Inside CHAPTERS.map — add to button content, after titleEn div:
const done = readCompletedLessons(ch.id).size;
const pct = Math.round(done / 24 * 100);
// Add after titleEn:
{done > 0 && (
  <div style={{ fontSize: 10, fontWeight: 700, opacity: 0.9, marginTop: 4 }}>
    {done}/24 · {pct}%
  </div>
)}
```
