## A. Angle research (5 bullet)

- **Left/right speaker alignment**: Duolingo Stories displays dialogue bubbles on alternating sides — protagonist/learner on LEFT, other characters on RIGHT. Character portrait (44×44 avatar) is always present in the bubble, not a text-label pill. This spatial cue lets children instantly know "who's talking" without reading a name.
- **Per-character portrait per bubble**: Each character has a dedicated icon (Lily, Junior, Zari, etc.) anchored to the bubble. Same portrait appears on all scroll-history entries for that character — visual continuity as user scrolls past.
- **Narrator ≠ character styling**: "Scene-setting" / background text is styled differently from character dialogue (italics, grey bg, no avatar). This prevents learner confusion between "story context" and "someone speaking."
- **Audio + text simultaneous**: Sentence text is visible the moment the bubble appears; audio auto-plays. No tap-to-reveal gate. After audio ends, the interface waits a configurable dwell (~2–3s) then advances. Manual "tap to continue" is always available.
- **Comprehension questions interspersed**: Every 2–4 narration lines, a question interrupts the scroll. The question type matches lesson goal (listen-mc, fill-blank, tap-pairs). This "narration + Q + narration" rhythm is the core Duolingo Stories UX loop.

## B. Pickup 現狀 (對齊這 angle, 5 bullet)

- **Single portrait for ALL speakers**: `NarrationRenderer` (renderers.tsx:193) hardcodes `calico-anchor.webp` (Mochi portrait) regardless of `q.speaker`. When grandma speaks, users see a cat avatar — brand confusion, especially for 8-12 children who rely on visual speaker cues.
- **No left/right alignment differentiation**: All bubbles render left-aligned regardless of speaker. `NarrationRenderer` (renderers.tsx:195-215) uses a single flex row with portrait always on the left. Zero conversational spatial cue.
- **SpeakerBadge pill is redundant noise**: `SpeakerBadge` (renderers.tsx:143-156) adds a text pill (🐱 Mochi / 👵 Grandma) above the bubble. Duolingo uses portrait only — no text label. Having both portrait (wrong) + label badge = double overhead, visual clutter.
- **NarrativeLine history loses speaker identity**: Past narration entries (LessonPage.tsx:195-210) render plain text + speaker icon with no character portrait. After 3+ lines of history, user cannot tell who said what — no spatial or portrait continuity.
- **Ch2-8 have ZERO narration entries** (confirmed master matrix line 31): 1,100+ questions but no `type:"narration"` entries, no `type:"listen-tf"` entries. The entire Duolingo Stories loop (narration → Q → narration) exists only in Ch1. All other chapters are pure MC drill — no storytelling UX at all.

## D. Top 3 actionable (S effort)

### D1 — Per-speaker portrait map (P1, S 20min)
**File**: `src/react-app/renderers.tsx:193`

```tsx
// Add after SPEAKER_META (renderers.tsx ~line 141):
const SPEAKER_PORTRAITS: Record<string, string | null> = {
  mochi:    '/mascots/calico-anchor.webp',
  grandma:  '/mascots/scene-grandma-icon.webp',
  hana:     '/mascots/scene-hana-icon.webp',
  narrator: null,
};

// In NarrationRenderer, replace hardcoded img:
const portrait = SPEAKER_PORTRAITS[q.speaker ?? 'mochi'] ?? '/mascots/calico-anchor.webp';
// ... and:
{portrait && <img src={portrait} width={44} height={44} alt="" style={{ borderRadius: 12 }} />}
```
Assets already exist: `scene-grandma-icon.webp`, `scene-hana-icon.webp`, `calico-anchor.webp`. Zero content changes.
