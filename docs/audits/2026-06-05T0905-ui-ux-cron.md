# Pickup UI/UX Cron — 2026-06-05 09:05 UTC

Today's angle: **Babbel SRS visual representation**
Source: https://www.babbel.com/magazine/babbel-review-manager

## A. Angle research (5 bullet)

- **Review Manager dashboard**: Babbel surfaces a persistent "Words for Review" counter on the home screen — always visible, acts as a re-engagement hook ("You have 14 words due"). Players can tap to launch a dedicated review session outside the normal lesson flow.
- **Mastery color tiers**: Each vocabulary item carries a visual state — grey (new), yellow (learning), blue (review due), green (mastered). Babbel shows this in a word-list notebook so players feel tangible progress toward mastery, not just lesson completion.
- **Post-lesson SRS feedback**: After every lesson Babbel's summary screen shows "X new words added to Review Manager," linking the lesson work directly to the SRS queue. The cause-effect loop is visible.
- **Review badge on lesson cards**: In the chapter map, lessons with pending reviews show a small badge ("3 to review"). Players can see at a glance which chapters need revisiting before advancing.
- **"This will come back" wrong-answer prompt**: When a player answers incorrectly, Babbel shows a subtle "Added to your review queue" label in the feedback panel — closing the SRS loop in-the-moment rather than silently.

## B. Pickup 現狀 (對齊這 angle)

- **SRS queue is completely invisible**: `srsReviewBatch()` injects 0–3 review questions silently at the start of each story chapter. The player never sees a queue count, never knows they have pending items, and has no way to intentionally trigger a review session.
- **Legacy key name still in production**: The SRS localStorage key is `wordwar.srs.kitten` — the pre-rebrand name. Rebrand to `pickup.*` namespace was never applied to this key. Low-level data integrity risk if tooling ever scans/migrates by prefix.
- **CompletePanel shows XP/Accuracy/Time but zero SRS signal**: Babbel closes the loop with "X words added." Pickup's `CompletePanel` (`LessonPage.tsx:~170`) calculates `correct` and `total` from `answerLog.current` but never reads `readSrsQueue().length` or surfaces how many questions entered/exited the SRS pool this session.
- **Tasks tab has no dynamic review card**: `TasksPage.tsx` shows streak + two static tasks. If a player has 8 questions in SRS queue, the Tasks tab gives zero indication — missing the re-engagement hook that Babbel's counter exploits.
- **MapView progress bar is lesson-count only**: `MapView.tsx` shows `ch.progress / ch.total` as a filled bar. No chapter-level SRS badge (Babbel shows these per lesson card).

### 1. P0 — SRS key rename (`wordwar.srs.kitten` → `pickup.srs.kitten`)

Change `'wordwar.srs.kitten'` → `'pickup.srs.kitten'`. In `readSrsQueue()`, add a one-time migration: if new key is empty, read old key, write to new key, remove old key. Zero UX change, zero data loss.

### 2. P1 — CompletePanel "X 題進複習" label
**File**: `src/react-app/pages/LessonPage.tsx`, `CompletePanel` function (~line 170)  
`answerLog.current` already has `isCorrect` per question. `const wrongCount = log.filter(a => !a.isCorrect).length`. Below the 3-stat row add:
```tsx
{wrongCount > 0 && (
  <div style={{ fontSize: 13, color: '#8b6f4a', margin: '0 0 12px', fontWeight: 700 }}>
    📥 {wrongCount} 題進複習庫 · see you next time
  </div>
)}
```
No new imports, no new state. Closes the SRS cause-effect loop that Babbel nails and Pickup currently leaves invisible.

### 3. P1 — Tasks tab dynamic SRS card
**File**: `src/react-app/pages/TasksPage.tsx`  
`import { readSrsQueue } from '../../data/storyKitten'`. Add `const srsCount = readSrsQueue().length;` at top. Insert a 4th task row:
```tsx
{ icon: '🔁', text: `複習 ${srsCount} 個待複習題`, done: srsCount === 0 }
```
Hides itself as `done` (✓) when queue is empty — no anxiety from zero. Surfaces the re-engagement hook Babbel uses to drive D7 retention.
