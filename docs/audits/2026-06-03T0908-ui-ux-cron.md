# Pickup UI/UX Cron — 2026-06-03 09:08 UTC

Today's angle: **Khan Academy progress visualization**
Source: https://www.khanacademy.org (unit/course mastery system, skill-tree map, progress bars, mastery-point counters)

---

## A. Angle research (5 bullet)

- **Mastery tiers (5-level)** — KA colors each skill node: Not Started (grey) → Familiar (blue) → Practiced (teal) → Mastered (green) → Proficient (dark green). Binary locked/unlocked is the minimum viable state; multi-tier mastery dramatically raises perceived value of repeated practice.
- **Unit-card progress bars** — Every course card on the dashboard shows a thin horizontal bar + "Mastery: 247/500 pts". Learners see both absolute progress and relative distance to the next milestone at a glance.
- **In-exercise live bar** — During a practice session KA shows a step-by-step segmented bar (5 correct → mastered), not just a text counter. The bar is the primary motivation to finish the session.
- **Lesson summary panel** — After a unit test KA shows a breakdown: ✅ Mastered (3 skills) / 🔄 Needs practice (2 skills). The qualitative label ("Mastered", "Practice more") carries more weight than a raw accuracy %.
- **Profile activity heatmap** — KA's coach dashboard shows a calendar heatmap of daily practice time. Even a simple streak calendar (past 7 days) communicates consistency better than a single "streak: 5" number.

---

## B. Pickup 現狀 (對齊這 angle)

- **Node state = binary** — MapPage nodes render done (star/green) or not-done (paw/brown), zero intermediate mastery tier. A lesson done at 50% accuracy looks identical to one done at 100%.
- **AlertsPage = dead data** — All 8 achievement badges hardcoded `earned: false`. The backing data (streak, XP, `readCompletedLessons`) already exists in localStorage; the evaluation logic simply was never wired, so the Achievements tab contributes zero sense of progress.
- **Chapter completion invisible from ChaptersPage** — 8 chapter list-buttons show title + lock icon only. No "8/24 lessons done" bar or counter. A returning user cannot see how far into a chapter they are without opening the map.
- **In-lesson progress = text only** — LessonPage shows `q4/14` as a small amber pill at top-right. KA (and Duolingo) treat the progress bar as the primary motivational affordance; Pickup's text counter does the bare minimum.
- **CompletePanel XP has no ceiling** — "XP +130" is shown after a lesson but the player sees no "you need 70 more XP for level 3" framing. KA always shows before/after within the level arc, making each session feel like tangible progress toward a visible goal.
