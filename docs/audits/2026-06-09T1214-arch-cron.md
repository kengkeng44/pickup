## D. Angle-specific risk — Dep graph circular detection

### Finding 1 (Cycle 1): `storyRegistry.ts` ↔ `userProfile.ts` — type-only, safe at runtime

**Chain:**
```
storyRegistry.ts  →  import type { UserPreferences }  →  userProfile.ts
userProfile.ts    →  import type { Culture, Style, Protagonist, Theme }  →  storyRegistry.ts
```

**Severity: LOW** — both sides are `import type`. TypeScript erases these at compile time; Rolldown/JS module evaluator never sees the cycle. No TDZ risk.

**Design smell**: Two data-layer modules are mutually type-coupled. `storyRegistry.ts` defines story/chapter types; `userProfile.ts` owns user state. Cross-import of *types* is acceptable short-term but signals that shared vocabulary types (`UserPreferences` fields driven by story-registry categories) belong in a third `src/data/types.ts` primitive file.

### Finding 2 (Cycle 2): `userProfile.ts` ↔ `storyTags.ts` — mixed value+type, latent TDZ risk

**Chain:**
```
userProfile.ts  →  import { STORY_TAGS }  →  storyTags.ts      ← VALUE import
storyTags.ts    →  import type { AbilityLevel }  →  userProfile.ts   ← type only
```

**Severity: MEDIUM** — the back edge (`storyTags → userProfile`) is type-only so no runtime cycle today. But:

1. `STORY_TAGS` is a large const object (lookup table for chapter tag metadata) initialized at module scope in `storyTags.ts`. `userProfile.ts` uses it in `inferPreferredTags()` (line 250).
2. If any future developer adds a single **value** import to the back edge (`storyTags.ts → userProfile.ts`), the cycle becomes a runtime circular module dependency. JavaScript ESM evaluation then hits a Temporal Dead Zone: whichever module loads second gets `undefined` exports from the not-yet-evaluated module.
3. With **Vite 8 / Rolldown** now in production (v8.0.14 confirmed), the bundler is *more verbose* about circular imports than Rollup was. Rolldown emits warnings where Rollup stayed silent — the current build only avoids warnings because TypeScript strips the type back-edge before Rolldown sees it.

**Root cause**: `AbilityLevel` (`'A0' | 'A1' | 'A2' | 'A2+'`) is defined in `userProfile.ts` but consumed in `storyTags.ts` for `CHAPTER_DIFFICULTY` mapping. The type belongs to the domain vocabulary shared between both modules, not owned by either.

**Cheapest fix** (S, 15 min): Extract `AbilityLevel` to `src/data/primitiveTypes.ts`. Both files import from the neutral primitive. Cycle broken with zero behavior change.

### Finding 3: No CI gate for circular deps

`package.json` has no madge/dpdm script. `.github/workflows/ci.yml` runs `validate-lessons.js` but nothing checks import cycles. A new developer can introduce a value-import back edge, pass CI, and ship a runtime TDZ bug.

**Cheapest fix** (S, 10 min): Add one line to `package.json` scripts:
```json
"check:circular": "madge --circular --extensions ts,tsx src/ && echo 'No circular deps'"
```
And add `npm run check:circular` step to `.github/workflows/ci.yml` before build.

### Finding 4: Vite 8 Rolldown circular warn parity

Project is already on Vite 8.0.14. Rolldown (the Rust-based bundler replacing Rollup) emits new warnings for circular imports that Rollup suppressed. Currently safe because TS erases the back edges before Rolldown sees them. Upgrading to Vite 9 (when it ships) may expose these more aggressively — the latent coupling is a migration time-bomb.

### Finding 5: Hub node concentration

`LessonPage.tsx` (19 imports) is the highest-import hub in the codebase — it imports from `data/`, `audio/`, `react-app/components/`, `store/`, and `scenes/`. Any circular dep introduced in any of those subtrees is one import away from pulling through LessonPage, the largest chunk (65 KB / 17.6 KB gzip).

## E. Top 3 P0 architecture risks

### P0-1 — Latent TDZ bomb in `userProfile.ts` ↔ `storyTags.ts` (Cycle 2)

`STORY_TAGS` (value import) from `storyTags.ts` + `AbilityLevel` (type-only back edge). Safe today; one careless value import on the back edge silently ships a runtime crash. With 110+ chapters being content-authored autonomously (cron agents), the probability of a developer touching `storyTags.ts` and adding a value lookup is non-trivial. **Fix: extract `AbilityLevel` to `src/data/primitiveTypes.ts`** (S, 15 min). Zero behavior change.

### P0-2 — No CI gate: circular dep can ship undetected

The entire circular dep detection relies on manual `npx madge` — not in `package.json`, not in CI. The existing `ci.yml` guards JSON schema (`validate-lessons.js`) and TypeScript types (`tsc --noEmit`) but has no import-graph guard. A circular value dep can pass all current CI checks and deploy. **Fix: add `"check:circular"` npm script + one CI step** (S, 10 min).

### P0-3 — tts.ts unlock listeners missing `{ once: true }` (R6 open, B.257 Decision Board #1)

`window.addEventListener('touchstart'|'click'|'pointerdown', unlockOnce, ...)` at lines 548–550 are permanent — they never detach after the first gesture. Every subsequent tap re-runs `unlockAudio()`. While `unlockAudio()` is idempotent (checks `isAudioUnlocked`), the triple-listener pattern accumulates during long sessions and creates unnecessary capture-phase overhead on every gesture in the app. **Fix: add `{ once: true }` to all three** (S, 5 min). Confirmed P0 from cockpit Decision Board #1.

## 🔬 Architecture Recommendation (對齊業界 2026)

### Industry scan (5 searches conducted)

1. **Vite 8 + Rolldown** (2026-03-12 shipped): Rust bundler replaces Rollup. Stricter circular import warnings. Pickup is already on v8.0.14 — exposure is live.

2. **ESM barrel file consensus (2026)**: Barrel files (`index.ts` re-exports) are increasingly flagged as anti-patterns — they're the primary source of accidental circular cycles in TypeScript projects. Pickup has no `src/data/index.ts` barrel — this is already the correct pattern.

3. **Vite plugin ecosystem**: `vite-plugin-circular-dependency` provides build-time throw-on-cycle enforcement. Budget: zero runtime cost, ~5 min config.

4. **dpdm-fast (Rust)**: For large TS repos, `dpdm-fast` (Rust port of `dpdm`) is 10–20× faster than madge for CI. Pickup's 87-file codebase is small enough that madge (1.2s) is fine.

5. **PWA 2.0 / edge pattern**: Static shell on CDN + service worker offline fallback + Cloudflare edge for config injection. Pickup's current architecture (Vite static → CF Pages + SW) already aligns with 2026 PWA best practice. No architecture change needed for deployment tier.
