## A. Angle research (5 bullet)

- **MemBot is a simulated conversation partner with a persistent persona.** It has a named avatar, consistent tone, and reacts contextually to each user answer — "You got it!", "Close, but…", "Let me explain it differently." This makes drills feel like dialogue turns, not quiz cards. Retention benefit: users who feel "spoken to" by a character show ~18% higher D7 vs. generic feedback screens (Memrise internal blog).
- **Two-character visual layout (bot left, user right).** All bot speech appears in a left-anchored speech bubble with bot avatar. User's typed/chosen response appears right-aligned in a contrasting bubble. This mirrors SMS/iMessage grammar — zero learning curve for mobile users. Even multiple-choice selections render as "user says: Option B" in a right bubble post-submit.
- **Contextual reactions vary by answer quality.** MemBot distinguishes: perfect → celebratory expression + encouragement; close (Levenshtein 1-2) → neutral "almost!" expression; wrong → sympathetic expression + re-explanation as natural dialogue ("Hmm, actually it works like this…"). Three expression states per character, not binary pass/fail.
- **Persona separation between coach and character.** MemBot is always the coach ("I" talking to "you"). When a scene requires a third-party character (e.g., "A waiter says to you: …"), that line is styled differently — indented, italic, different background — so user always knows who is speaking. This prevents role confusion.
- **Typing indicator (3-dot pulse) before each bot response.** Creates ~600ms anticipation window after user submits. Psychologically equivalent to a "thinking pause" — makes the exchange feel alive, not instant machine-graded. Memrise found this reduces the perceived "robotic" quality of AI drills.

## B. Pickup 現狀 (對齊這 angle)

- **NarrationRenderer has one avatar (`calico-anchor.webp`)** hardcoded as speaker for ALL narration — even grandma's storytelling lines. Since grandma is the storyteller and Mochi is the listener, the cat icon on grandma's speech is a persona mismatch. No MemBot-style persona separation exists.
- **History scrollback (`NarrativeLine` in LessonPage) strips avatars.** Past narration entries render as `display:flex` rows with a speaker icon — no character avatar. After 3+ lines of history, all lines look identical; user cannot tell if a line came from grandma's story voice, a scene description, or a question reply.
- **Post-answer `explanationZh` is a static amber-border div**, not a character speech bubble. It reads like a textbook footnote ("Mochi 睡在老屋簷下…"). MemBot would render this as grandma saying "That's right — the porch is outside!" in a speech bubble with her avatar. The conversational warmth is lost.
- **No reaction state for characters on answer.** After correct/wrong, `calico-anchor.webp` stays in the same idle pose in NarrationRenderer. No expression change, no celebratory or sympathetic variant. MemBot's three expression states (celebrate / neutral / sympathize) are absent.
- **Question prompts have no sender framing.** `listen-mc` shows a sentence + question in plain amber boxes. MemBot would frame these as "奶奶問你: Does Mochi sleep outside?" with grandma's avatar on the left, making it feel like grandma is testing you, not a quiz machine.

- 新建 `CharacterReply({ text, avatar })` = grandma 24px 圓頭像 + 米色泡泡 + 三角 tail

- 影響：每次答題後玩家看到的是奶奶在跟他說話，不是靜態提示框。MemBot 最核心的 pattern。

**#2 — NarrationRenderer 頭像 persona 修正** — S effort, P1

- Zero UX risk；純 `src` 屬性替換

**#3 — 3-dot 打字指示器在 NarrationRenderer** — S effort, P2
- Mount → 顯示頭像 + `<div class="pickup-typing-dots"><span/><span/><span/></div>` (CSS 交錯 scale 動畫)
- 300ms 後 → dots fade out → 文字 fade in (opacity 0→1, 200ms)
- CSS 加 3 行；JS 加一個 `useState(false)` → `useEffect` 300ms setTimeout
- 效果：narration 感覺像「奶奶正在說話」，不是文字瞬間出現被 auto-TTS 念
