## A. Angle research (5 bullet)

- **Mastery system with 5 tiers**: Attempted → Familiar → Proficient → Mastered → Legendary. Each skill shows a colored arc or ring — learner always knows exactly where they stand, not binary locked/unlocked.
- **Unit progress bars with counts**: Every unit header shows a horizontal bar + "X / Y exercises" count. At a glance from the course overview, learner sees which units are 0%, 50%, 100% done.
- **Dashboard course snapshot**: Dedicated view of all units in a course, each with a progress fraction. Learner can see "I'm 2/6 done with Algebra 1" without entering each unit.
- **Points + recent activity feed**: Running energy-point total is always visible with a micro-history ("earned 300 points today"). Learner gets reinforcement even for small sessions.
- **"Get back on track" nudge**: Khan surfaces a banner when a streak or goal is broken — not punitive, just a low-friction "continue where you left off" CTA anchored to the specific skill.

---

## B. Pickup 現狀 (對齊這 angle)

- **MapPage** (Chapter Map): Shows 24 paw nodes per chapter — done nodes turn olive+star, locked are greyed. BUT the chapter header card has zero progress count ("Section 1 · 第1章" with title only). Learner cannot see "12/24 done" without counting nodes by eye.
- **ChaptersPage** (31 chapters list): Each chapter is a full-width button with `titleZh + titleEn` — NO progress bar, NO X/Y count. A returning user with progress in Ch3+Ch7 has zero signal which ones are partially done vs untouched.
- **AlertsPage** (Achievements): 8 achievement cards are ALL hardcoded `earned: false`. Streak ≥3, streak ≥7, XP ≥1000, first lesson — none are wired to real localStorage data. The motivation loop is broken at the achievement layer: you earn things, you see nothing earned.
- **ProfilePage stats**: 4-stat grid (Streak / XP / Coins / Crown Level) — no "lessons completed", no "chapters finished", no "lessons this week". Khan's stats page shows relative progress (this week vs last week). Pickup shows raw numbers with no context.
- **TasksPage goal accuracy**: "正確率 ≥ 80%" item is hardcoded `done: false` with "(待 session log)" — never resolves. Dead goal card that always shows ○ → frustrates rather than motivates.
