## A. Angle research (iOS Health chart UX — 5 bullets)

1. **Activity Rings (三環) = glanceable daily goals**. Apple's three concentric arc rings (Move/Exercise/Stand) let users understand "how close to goal" in <2 seconds. Each ring has a distinct color, fills as progress accumulates, and "wraps around" if goal is exceeded. This pattern is proven for daily habit loops — kids respond strongly to the visual feedback of a ring closing.

2. **Sparkline micro-charts beside every stat**. iOS Health never shows a raw number without a 7-day bar chart / sparkline next to it. The combination of "number + tiny chart" communicates both current state AND trend — far more actionable than a number alone. In 2026 wellness apps this is table-stakes.

3. **Weekly narrative summary card**. Health's "Summary" tab opens with a narrative sentence: "This week you averaged 8,420 steps/day, 12% above your goal." This gives the data a story instead of a spreadsheet. Education apps (Duolingo Year in Review, Khan Academy weekly email) have copied this pattern successfully.

4. **Progressive disclosure — tap → drill**. Every metric card in Health is tappable: tap the Streak number → see full 30-day calendar. This respects screen real estate (summary view = glanceable) while rewarding curious users with depth. Scroll-depth and engagement both increase.

5. **Color-coded trend arrows**. Health shows a small ↑12% or ↓5% arrow next to each stat — immediate period-over-period context. For kids, this is motivating: seeing "↑ from last week" creates pride. Apps that show trend arrows see higher D7 retention (per Diversido 2026 UX report).

## B. Pickup 現狀 (對齊 iOS Health angle — 5 bullets)

1. **ProfilePage stats = raw numbers with zero visualization** (`src/react-app/pages/ProfilePage.tsx:83`). Four numbers in a 2×2 grid: Streak 🔥, XP, Coins, Crown Level. No chart, no sparkline, no trend arrow, no history. Users have no idea if they're improving or plateauing.

2. **TasksPage daily goals = emoji + checkmark only** (`src/react-app/pages/TasksPage.tsx:38`). Three goals (lesson, streak, accuracy) rendered as plain `div` rows with `✓` or `○`. iOS Health-style rings would make these feel tactile and rewarding — especially for 8-12 year old players who are the target audience.

3. **AlertsPage achievements = ALL hardcoded `earned: false`** (`src/react-app/pages/AlertsPage.tsx:2-9`). Every achievement badge is permanently greyed out for every user. The data layer (`readStreak()`, `readXp()`, `readCompletedLessons()`) already exists and is imported in ProfilePage — AlertsPage just never wires it. This is a **P0 bug**: every user sees all achievements as unearned regardless of their real progress.

4. **No sparkline or trend visualization anywhere**. XP and Coins are tracked in localStorage (`src/data/xp.ts`, `src/data/coins.ts`) but only as a cumulative total. There is no daily snapshot series, so trend charts are impossible without a new `weeklyStats.ts` data layer.

5. **No weekly narrative summary**. Pickup has a GrandmaRecommendCarousel (`src/react-app/components/GrandmaRecommendCarousel.tsx`) and a TomorrowQueue banner, but nowhere does the app say "This week Mochi listened to 5 stories." The data for this exists (completedLessons per chapter in runStore) — it's just never surfaced as a human sentence.

## D. Top 3 Actionable (S effort)

### D1 ⚠️ P0 — AlertsPage 接真實 earned 資料 (S · 20min)
**File**: `src/react-app/pages/AlertsPage.tsx`  
**Fix**:
```tsx
// 在 AlertsPage 頂部加 imports:
import { readXp } from '../../data/xp';
import { readStreak } from '../../data/streak';
import { readCompletedLessons } from '../../store/runStore';

// 在組件 body 內:
const xp = readXp();
const streak = readStreak();
const ch1done = readCompletedLessons(1).size;
const hasAnyLesson = ch1done > 0;
const ch1full = ch1done >= 24;
let completedAllCh = true;
for (let c = 1; c <= 8; c++) { if (readCompletedLessons(c).size < 24) { completedAllCh = false; break; } }
const ch2unlocked = readCompletedLessons(1).size >= 24; // ch2 unlocks after ch1

// ACHIEVEMENTS 改為 computed:
const ACHIEVEMENTS = [
  { ..., earned: hasAnyLesson },       // 第一步
  { ..., earned: streak >= 3 },        // 三日連擊
  { ..., earned: streak >= 7 },        // 七日狂熱
  { ..., earned: false },              // 完美主義 (sessionLog not tracked yet)
  { ..., earned: ch2unlocked },        // 探索者
  { ..., earned: ch1full },            // 說書人
  { ..., earned: xp >= 1000 },         // 神射手
  { ..., earned: completedAllCh },     // 畢業生
];
```
**Why now**: Every user sees all badges as unearned forever. A user who has played for 3+ days and maintained their streak never sees the "三日連擊" badge light up. This is a pure data-wiring miss with a 20-min fix.

### D2 P1 — TasksPage 3 daily goals → SVG ring (S · 30min)
**File**: `src/react-app/pages/TasksPage.tsx`

Replace each goal row with a compact `RingGoal` component:
```tsx
function RingGoal({ icon, label, fraction, color }: { 
  icon: string; label: string; fraction: number; color: string 
}) {
  const R = 18, C = 2 * Math.PI * R;
  const filled = C * Math.min(fraction, 1);
  return (
    <div style={{ display: 'flex', alignItems: 'center', gap: 12, padding: '12px 0' }}>
      <svg width={44} height={44} viewBox="0 0 44 44">
        <circle cx={22} cy={22} r={R} fill="none" stroke="#e0d0b8" strokeWidth={4} />
        <circle cx={22} cy={22} r={R} fill="none" stroke={color} strokeWidth={4}
          strokeDasharray={`${filled} ${C}`} strokeLinecap="round"
          transform="rotate(-90 22 22)" />
        <text x={22} y={27} textAnchor="middle" fontSize={16}>{icon}</text>
      </svg>
      <span style={{ fontSize: 14, color: '#3c2a1c', fontWeight: 600 }}>{label}</span>
    </div>
  );
}
```
Use `fraction = hasCompletedAnyLesson ? 1 : 0` for lesson goal, `fraction = Math.min(streak / 7, 1)` for streak, `fraction = 0` for accuracy (deferred). Zero new deps.
