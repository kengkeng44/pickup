## A. Angle research (5 bullet)

1. **5-minute countdown is the hero mechanic.** Drops leads with "5 minutes a day" as the *entry CTA itself* — not buried in onboarding, but shown before the user commits. The countdown ring is prominent throughout the session. "Time's up" = graceful end, not a fail state. Based on Csikszentmihályi's flow research: a tight time box creates hyper-focus, not stress.

2. **Content is pre-engineered to fit 5 min exactly.** Each vocabulary set = ~15 words × ~20s each = 5 min. Session length is *predictable*: the user always knows the session will end when the timer runs out. There is no variable "this lesson has 5 questions, this one has 15."

3. **Pre-session commitment display.** Before tapping a topic, Drops shows the session type + implied time cost. The user signs up for a known commitment ("5 min, vocabulary, farm animals"). There is no ambiguity about what they're getting into.

4. **Parent-permission UX.** The "5 minutes a day" slogan is specifically engineered to get *parental buy-in* for child users — it is the exact phrase that works in a parent-child negotiation. Common Sense Media's review of the children's version (Droplets) explicitly notes this as a key adoption driver.

5. **Post-session celebration = time invested, not score.** Drops celebrates "you used your 5 minutes" as much as what you learned. The completion screen highlights the elapsed time as an achievement, not just correctness. Daily habit is the KPI, not accuracy.

## B. Pickup 現狀 (對齊 Drops session pacing angle, 5 bullet)

1. **MapNode 無任何時長信號.** `MapPage.tsx:687` — `ariaLabel` 只有 `l.storyBeat + locked/current`。視覺上 node 只有 paw icon。用戶點前完全不知道「這顆節點要花幾分鐘」。Drops 在選題目時就展示時間。

2. **LessonPage header 無預估時長.** `LessonPage.tsx:104` — header 有 ✕ + progress bar + 🔇。進度條顯示當前進度但不顯示剩餘時間或「預估還有 X 分鐘」。Drops 的倒計時環是 session 的心跳。

3. **課程長度不一致，用戶無法建立時間預期.** Pickup 每個 lesson 5-15 Q（依 segment type），`25s/Q` 均速下 = 2–6 分鐘不等。對 8-12 歲兒童而言，不可預期的時間長度 = 進入障礙（家長問「要多久？」無法回答）。

4. **TasksPage 每日目標無時間框架.** `TasksPage.tsx:43` — 「完成 1 個 lesson · Complete 1 lesson」沒有 `≈ 3-5 分鐘` 的時間錨。兒童客群的核心 adoption 邏輯就是「奶奶睡前5分鐘故事」— 但 UI 沒有落地這句話。

5. **CompletePanel TIME stat 太晚出現，且視覺地位低.** `LessonPage.tsx:327` — `<Stat label="TIME" value={timeStr}.../>` 出現在課後完成畫面，三欄之一（XP / ACCURACY / TIME）。Drops 把「用了 5 分鐘」當 hero 顯示，Pickup 把時間藏在第三格。時間沒有成為每日習慣的 trophy。

## D. Top 3 actionable (S effort)

### D1 — MapNode ariaLabel + header 加 `estimateLessonMinutes` 時長 (P1, S 20min total)

**Files**:
- `src/react-app/pages/MapPage.tsx:687` — ariaLabel
- `src/react-app/pages/LessonPage.tsx:104` — lesson header

`estimateLessonMinutes(n: number): number = Math.ceil(n * 22 / 60)` (22s/Q = 3s auto-advance + ~19s read/answer average, conservative for A2 children).

MapPage 改：
```ts
// line 687 現行:
const ariaLabel = `${l.storyBeat ?? `Lesson ${l.lessonInChapter}`}${unlocked ? '' : ' (locked)'}${isCurrent ? ' (current)' : ''}`;
// 改成:
const estMin = Math.max(1, Math.ceil((l.questions as unknown[]).length * 22 / 60));
const ariaLabel = `${l.storyBeat ?? `Lesson ${l.lessonInChapter}`} · ~${estMin}min${unlocked ? '' : ' (locked)'}${isCurrent ? ' (current)' : ''}`;
```

LessonPage header 改（lesson load 後取 `lesson.questions.length` 算一次 `estMin`，顯示在 progress bar 右）：
```tsx
// after progress bar div, add:
<span style={{ fontSize: 11, color: '#a08060', fontWeight: 700, flex: '0 0 auto' }}>~{estMin}m</span>
```
