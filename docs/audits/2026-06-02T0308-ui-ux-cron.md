Today's angle: **Babbel SRS visual representation**

## A. Angle research (5 bullet)

- **Review Manager widget on Home**: Babbel shows "X words due for review" as a prominent card on the home screen — a number badge + "Review now" CTA. Users never wonder when to come back; the system tells them.
- **Per-word mastery tiers**: each vocabulary item carries one of 4 states (New / Familiar / Learned / Mastered) shown with a colored ring icon. Tap any word in "My Vocabulary" to see its tier + next review date.
- **Post-lesson SRS feedback panel**: after every lesson, Babbel shows "Y new words added to review queue" with a mini word-cloud. Users see their queue growing, which creates investment.
- **Review countdown + next-session timing**: Babbel's Review Manager shows exactly when the next batch is due ("Next review: in 4 hours"). This is the primary retention trigger for D1-D7 — users return specifically for their review slot.
- **Cumulative vocabulary counter**: "You know 347 words" is always visible in the profile. This single number is Babbel's most motivating long-term stat — it grows with every correct answer, not just on level-up.

## B. Pickup 現狀 (對齊這 angle)

- **HUD XP & Coins hardcoded 0**: `MapPage.tsx:161-166` sets `const xp = 0; const level = 1; const coins = 0` — never reads from localStorage. `readXp()` and `readCoins()` exist in `src/data/xp.ts` / `src/data/coins.ts` but are never imported in MapPage. Every session the crown shows "L1" and coins show nothing, even for returning players.
- **ProfilePage stats all show "—"**: `ProfilePage.tsx:41-43` renders `value="—"` for XP, Coins, and Level. The data layer is complete (readXp / readCoins / levelForXp all exist) but the component never calls them.
- **SRS queue completely invisible**: `readSrsQueue()` in `storyKitten.ts:381` returns the queue of wrong-answer IDs, but zero UI surface in React app exposes this. User has no idea how many items are pending, when they'll see them, or that the system even exists.
- **TasksPage goals always `done: false`**: `TasksPage.tsx:20-29` hardcodes three tasks with `done: false`. "完成 1 個 lesson" is never checked against `readCompletedLessons()`. Even after completing a lesson, Tasks tab shows zero progress.
- **Achievements all hardcoded `earned: false`**: `AlertsPage.tsx:1-10` defines 8 achievements, all with `earned: false` — no condition is ever evaluated. A user who has completed Ch1 or earned 1000 XP sees no earned badges.

## D. Top 3 actionable (S effort)

### 🥇 Fix 1: Wire MapPage HUD to real XP/Coins/Level (P0)
**File**: `src/react-app/pages/MapPage.tsx:161-166`
**Change**: Replace 3 hardcoded constants with calls to existing data layer:
```ts
import { readXp, levelForXp, levelProgress } from '../../data/xp';
import { readCoins } from '../../data/coins';
// inside component:
const xp = readXp();
const level = levelForXp(xp);
const coins = readCoins();
const xpFraction = levelProgress(xp).fraction;
```
Then pass `xpFraction` to the Crown HudIcon `progress` prop (line 191).
Impact: returning player immediately sees real level, coin count, and XP bar — the single highest-signal "you are progressing" moment.
