# Pickup UI/UX Cron Audit — 2026-06-01 16:31 UTC

## A. Trend research summary

- **Duolingo World Character lip sync via Rive visemes (Jan 2025)**: 20+ mouth shapes per character, each mapped to phoneme timing from speech recognition. State Machine drives idle / talking / correct / wrong states at runtime. For Pickup: calico-anchor + grandma are currently CSS `pickup-mascot-bob` only — no reactive state on answer events. Source: https://blog.duolingo.com/world-character-visemes/
- **Memrise AI Buddies (May 2025)**: four specialized AI personas (Grammar / RolePlay / Translator / Culture) replaced broad pronunciation practice. Chat-first UX where user can steer conversation. Signals the industry shift from quiz-only to mixed quiz + conversation. Pickup is quiz-only; no conversational surface yet. Source: https://www.memrise.com/blog/introducing-membot
- **925studios "Play First, Profile Second" (March 2026)**: Duolingo gets users to 10% fluency feeling (first lesson, XP earned, near-completion) before prompting profile/signup. Pickup's current boot goes directly to MapPage — user sees 24 locked nodes with no onboarding lesson friction. Source: https://www.925studios.co/blog/duolingo-design-breakdown
- **orbix.studio retention patterns (2026)**: "accomplish core task within 60 seconds" focus loop + celebrate success immediately after each answer = highest D1 retention signal. LessonPage auto-advance timing is 2.5–5s post-answer; no explicit celebration before advance. Source: https://www.orbix.studio/blogs/ux-patterns-for-mobile-app-retention
- **Rive state machine for mascots (grayhat 2025 case study)**: interactive mascots (idle / correct / wrong / celebrate states) improved simpleclub DAU +62% and 30-day retention +28%. Source: https://www.grayhat.studio/echo/our-struggles-and-wins-with-rive-2025/
