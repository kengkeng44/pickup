Today's angle: **LingQ known-words counter (vocabulary mastery)** (angle #11)
Source: https://www.lingq.com/en/learn/zh/web/reader/ + LingQ app UX teardowns (2025)

---

## A. Angle research (5 bullet)

- **Known-words counter is the hero metric**: LingQ's home screen leads with a giant "Known words: 3,248" number. Users obsess over it the way Duolingo users obsess over streaks. Each word has a status tier: Unknown → Learning 1/2/3 → Known (4+), color-coded (blue = new/unknown, yellow = learning 1-3, white/light = known).
- **Passive vocabulary growth via tap**: Every tap on an unknown word in a story promotes it from Unknown → Learning 1. The act of checking translation = marking it "noticed." This creates a virtuous loop: read more → tap more → word count grows → read more.
- **"My Words" list**: A dedicated vocabulary bank (all words at each status tier) is a core power feature. LingQ users maintain their word lists; it's a tangible artifact of time invested.
- **Daily goal = vocabulary, not lesson count**: LingQ's home dashboard shows "Goal: 25 LingQs today (3/25)" where 1 LingQ = 1 new word looked up. Lesson-centric goals (complete 1 chapter) don't directly signal vocabulary growth.
- **Milestone levels**: LingQ maps word count to CEFR-like milestones (e.g. 5,000 known = functional B1). This gives learners a north-star target beyond "finish chapter 8."

---

## B. Pickup 現狀 (對齊這 angle)

- **WordHint exists but is stateless**: `src/ui/WordHint.ts` has 771-word `word-hints.json` dictionary and a tap-to-translate tooltip, but ZERO data is written to localStorage when a word is looked up. Every session starts with amnesia — Pickup has no idea which words a user has tapped once vs. 50 times.
- **Profile stats grid = game metrics only**: ProfilePage shows Streak 🔥 / XP / Coins / Crown Level — all gamification numbers. Zero vocabulary metrics (words seen, words looked up, unique cloze targets completed). A child who finished Ch1 can't see "I learned 47 new English words."
- **Cloze targets are vocabulary gold, untapped**: Every `cloze` question has exactly 1 target word (the blank). Ch1 alone has ~110 Q across 24 lessons. 30 chapters × ~110 Q ≈ 3,300 vocabulary exposures. None of these are tracked as "vocabulary met" or surfaced to the user.
- **TasksPage daily goals are lesson-centric**: "完成 1 個 lesson · Complete 1 lesson" and "連續學習 1 天 · Streak active." LingQ would surface "Today: tapped 3 new words" alongside. No vocabulary growth signal in daily tasks.
- **SRS lite exists but invisible**: Per CLAUDE.md, wrong-answer words enter a review queue. But there's no UI for "words in review queue" count, no "you've seen this word 3 times" feedback, no vocabulary bank page. SRS is silent infrastructure.

## D. Top 3 actionable (S effort)

**#1 — WordHint tap tracking (S, 30 min)**
The single unblocking data layer. `wireSentenceHints()` in `src/ui/WordHint.ts` fires on every word tooltip show (~line 90 in the pointerdown handler). One `try/catch` localStorage write:
```ts
// inside the tooltip-show branch:
try {
  const today = new Date().toISOString().slice(0, 10);
  const raw = localStorage.getItem('pickup.hint_taps') ?? '{}';
  const rec: Record<string, string[]> = JSON.parse(raw);
  rec[today] = [...new Set([...(rec[today] ?? []), word])];
  localStorage.setItem('pickup.hint_taps', JSON.stringify(rec));
} catch {}
```
This unblocks all downstream vocab stats (#2, #3, P2 P3 rows above) with zero UX change.

**#2 — Profile "Words Seen" stat card (S, 45 min)**
In `ProfilePage.tsx` stats grid (currently 4 cells), add a 5th `<Stat>`:
- Derive from `Array.from({length:8}, (_,i)=>readCompletedLessons(i+1).size).reduce((a,b)=>a+b, 0)` × estimated 4 vocab words per lesson = a rough "words practiced" count.
- Label: `詞彙 Words` / value: e.g. `96` (24 lessons × 4 words).
- Shows children/parents "I've practiced 96 English words" — more meaningful than XP alone.

**#3 — TasksPage vocabulary daily goal row (S, 30 min, after #1)**
Replace the hardcoded `accuracy ≥ 80%` row (which is always `done: false`) with:
```ts
{ icon: '📖', text: '查 3 個生詞 · Look up 3 new words', 
  done: getTodayHintTapCount() >= 3 }
```
where `getTodayHintTapCount()` reads from `pickup.hint_taps[today].length`. Gives vocabulary growth a daily ritual presence alongside streak.
