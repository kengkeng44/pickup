## A. Angle research (5 bullet)

- **Core loop: Listen → Speak → Instant feedback.** Speak's entire session is built around a large (52×52) circular mic button that pulses rhythmically, clearly signaling "it's your turn to speak." The visual state machine has 4 distinct states: idle (grey ring), listening (green animated ring + waveform), processing (spinner), result (green ✓ / red ✗ + phoneme highlighting).
- **"Is audio playing?" is always visible.** When TTS plays model speech, the speaker button animates a sound-wave SVG (3 bars, staggered scale). When audio ends, bars freeze. Users never have to guess whether audio started, finished, or silently failed.
- **Replay affordance is paramount.** Every audio surface has a "Tap to replay" micro-label below the speaker icon for first-time users; the label fades after the user has successfully replayed once (one-time education, not permanent clutter).
- **Audio failure degrades gracefully with visible feedback.** If TTS load fails (timeout or network), Speak shows a "⚠ Audio unavailable — tap to retry" state on the speaker button, not a silent gap. The lesson does not auto-advance if audio never played.
- **Chain audio is sequenced with visual breath.** When sentence audio ends and a follow-up prompt begins (e.g., "Now you try"), Speak shows a 300ms pause indicator (subtle ripple) so users understand a second clip is coming rather than assuming audio ended.

## B. Pickup 現狀 (對齊這 angle)

- **SpeakerBtn has a single static visual state.** `src/react-app/renderers.tsx:84-93` — opacity always `0.7`, no className, no animation. Idle, playing, loading, and failed states are visually identical. Zero "is it working?" signal.
- **No `onStart` callback in `tts.ts`.** `speak()` exposes `onEnd` (line 264) but no `onStart`. Renderers cannot know when audio actually begins playing vs. when `speak()` was called. A 400ms MP3 fetch gap is silent and invisible to the user.
- **`pickup-speaker-pulse` CSS animation exists but is orphaned in React.** `src/style.css:471-487` defines `.pickup-listen-btn` + `pickup-speaker-pulse`. The new React `SpeakerBtn` component (renderers.tsx:84) has no className, so the pulse never fires in lesson screens. Legacy PlayScene only.
- **TypeWhatYouHear correct-answer reveal has no replay speaker.** After wrong answer, `正解: {en}` text shown at line 375-380 but no SpeakerBtn — user cannot hear the correct pronunciation. Speak always provides "hear model answer" after wrong attempt.
- **Chain audio (sentence → question) has no visual breath indicator.** `ListenMcRenderer:266-269` — sentence plays, 400ms gap, then question plays. User sees no indication a second clip is queued; a tap on the speaker mid-chain cancels the queue silently.
