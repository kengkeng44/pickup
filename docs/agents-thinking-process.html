各 persona 開場意見

陶敘事

我先 push back 三件事。第一,桃太郎放 Ch2 太早。桃太郎是「主角集結同伴」的累積結構,但這時 user 才剛理解「奶奶說故事」的外層框架,馬上塞一個有四個人物的累積劇情會打亂節奏。Ch2 應該是更 intimate、更 close-up 的故事。第二,Baba Yaga 放 Ch6 過於黑暗,跟「下班逃逸」核心衝突。第三,葉限放最後當收尾很妙,但前面沒有任何中華 motif 鋪墊,Ch8 出現會像外掛。

陳品牌

我同意陶的「Baba Yaga 違反下班逃逸」。但桃太郎放 Ch2 是對的 — 它是日系最熟、最暖、最有「萌寵集結」的故事,跟 Mochi+Hana 「貓狗朋友」基線剛好同構,user 會自動帶入。葉限收尾我同意保留 — 它本來就是隱性中華梗,在 Ch7-8 雙語 code-switch 才會被注意,提早鋪沒必要。

林教學

從字彙叢集看八章的 vocab spread:

  • Ch2 桃太郎 → animal companions、quest verbs(fight, sail, give)、food(rice ball)
  • Ch3 醜小鴨 → identity(ugly, beautiful)、emotion(sad, alone)、seasons
  • Ch4 龜兔賽跑 → comparative(fast, slow)、dialogue verbs
  • Ch5 駱駝駝峰 → desert、why-because、body parts
  • Ch6 Baba Yaga → forest、scary、house parts
  • Ch7 六隻天鵝 → siblings、silent、weave
  • Ch8 葉限 → river、wish、fish、moon

叢集分佈我可以接受。但「龜兔賽跑」既是 Ch2 Aesop side、又是 Ch4 main 嗎?那 user 在 Ch2 已經聽過縮減版,到 Ch4 重看完整版會覺得冗 — 教學上 vocab 過早曝光,reuse rate 不夠高。

游體驗

我最在意的:Ch1 是雨夜小貓(meta-anchor,主角自己的故事),Ch2 馬上跳到桃太郎 — 從「我自己」到「別人的故事」,情感連結會突然中斷。user 會問:「等等,那隻雨夜小貓最後怎麼樣了?」答不出來他就走。建議 Ch2 留一個關於 Mochi 自己的小故事,做情感過渡。

梁語言

A2 角度,我建議 Ch2 走簡單敘事結構(not Kipling 第二人稱、not 對話體 alternating)。Ch1 已經是第一人稱了,Ch2 該換到第三人稱簡單過去式,讓 user 適應 narrative tense。桃太郎剛好是第三人稱,符合。

第 2 輪 — 互動模式:奶奶問 Mochi/Hana 怎麼設計

議題:用戶最 critical 的要求:題目中要穿插「奶奶問 Mochi/Hana」+「Mochi/Hana 視角解讀」橋段。要怎麼落地進 5Q/lesson 的結構,不增加題目數又不犧牲 vocab 教學密度。

各 persona 開場意見

陶敘事

這正是我最想做的!我提議三層互動模式:

  • A. 奶奶問句(Grandma Asks):奶奶對著貓狗問一個 A2 問句,e.g. "{catName}, do you think the boy is brave?"
  • B. Mochi/Hana 視角回應(Child Echo):回應該童話事件,用幼稚化的觀點 — Mochi 害怕但 Hana 樂觀
  • C. 後設笑點(Meta Beat):奶奶笑著糾正或補充

這三層自然形成一個 mini scene,我建議每 lesson 的 Q5 變成這種互動題(取代原本的純 comprehension)。Q5 用 listen-comprehension 體裁:奶奶問「你覺得 X 嗎?」,選項是貓狗可能的回答。

林教學

我支持,但要設條件。不能每 lesson Q5 都是互動題 — 那會變模板感,也會擠壓 comprehension 真正測理解的題量。

建議:

  • 每章 24 lessons 裡,抓 6-8 個「Interaction Beat」lessons(約 1/3 比例)
  • 分佈在 outer prologue(2-3)、main story 中段(2-3)、aesop 後(1-2),review 不算
  • 互動題仍然用既有題型(listen-comprehension / listen-emoji / tap-pairs),不增加新題型,UI 不用改
  • 互動題仍然教 vocab,只是 framing 換成「Grandma's question」+「Mochi/Hana's interpretation」
游體驗

我覺得這設計超讚的,因為它解決了一個我一直擔心但沒講的問題:「inner story」的 user 是空白聽眾。當奶奶說桃太郎時,user 在哪?他只是按題目。但如果 Mochi 跟 Hana 變成「替 user 反應」的代理,user 透過貓狗的天真解讀來投射自己 — 這就是 Pixar Soul 那種觀點切換的魔法。

我加碼一個 idea:讓貓狗給「不同」的解讀。Mochi 是愛哭鬼但堅韌,所以她會說「桃太郎好可憐,離開奶奶」;Hana 是樂觀柴犬,所以她說「我也想去打鬼!」奶奶笑了。這種雙視角讓 user 看到「同一件事可以兩種方式感受」,治癒感拉滿。

陳品牌

游講到我心坎裡了。愛哭鬼 Mochi + 樂觀 Hana 的雙視角,是 Pickup 從 v1.9 累積到現在最有資產感的設定。我們應該把這個設定壓到每一個互動題裡。

但我要 push back 一個細節:奶奶不能像老師一樣解釋。她應該像「在跟兩隻孩子聊天」 — 微笑、簡短、不一定有對錯。e.g.

奶奶:「桃太郎為什麼出發?」
Mochi:「因為他媽媽叫他不要回家。」(誤讀但 cute)
Hana:「因為鬼島有點心可以吃!」(樂觀曲解)
奶奶:「都不太對,但你們都很努力想。」

這比「正確解釋」更有品牌調。

梁語言

我提醒一個技術點。如果互動題用 listen-comprehension,「奶奶問」要用 audio 念出來 — 但目前 TTS 都是同一個聲音。我們不能假裝有兩個 voice,所以互動題的 sentence 應該明示「Grandma asks:」開頭,讓 user 從文字看到 speaker 切換。e.g.

  • Q sentence: Grandma asks, "Mochi, are you scared too?"
  • Options: "Yes, I cry a little" / "No, I am brave" / "Maybe later" / "I don't know"
  • 正解可以是「Yes, I cry a little」 — Mochi 的性格

這樣 A2 vocab 還教到,人物性格也鞏固。

第 3 輪 — 題目優先,場景跟隨

議題:用戶明確說「以劇情題目為優先」。意思是先寫題目+互動,再回頭設計章節背景畫面。這對視覺團隊和文案團隊的工作順序有什麼影響?

陶敘事

這正是我一直想要的工作流!過去產品的痛點:文案被視覺先綁死,寫題目要遷就場景設計。改成題目先決定,每章「主場景」就是該章劇情高峰所在的位置。e.g.

  • Ch2 桃太郎:題目高峰在「鬼島之戰」,主場景就是鬼島海岸夕陽
  • Ch3 醜小鴨:題目高峰在「照鏡子看見自己變天鵝」,主場景就是湖面倒影黎明
  • Ch5 駱駝駝峰:題目高峰在「精靈賜駝峰」,主場景就是沙漠日出

每章畫一張主視覺就夠,其他 lesson 用 Mochi+Hana+奶奶的 outer scene 重複(可省 60% 美術工)。

游體驗

我支持,但要補一個 UX 細節。Duolingo 9 年資料告訴我們「同一個背景看 24 lesson 會無聊」。建議:

  • 每章一張主場景圖(內故事高峰)
  • 外層 outer scene 用 3 種光照變化(奶奶開書時黃昏 / 故事高潮時月亮 / Goodnight 時燭火將熄)
  • 用同一張 base 改光色就能做出,成本接近零
陳品牌

我加一條 brand 守則:每章主視覺都要看得到 Mochi 跟 Hana 是「在聽」。即使是鬼島場景,角落也要有奶奶的書頁邊緣、或月光下兩隻小動物剪影。這提醒 user「這是故事,不是現實」 — 維持治癒感不被內 story 的緊張感壓垮。

林教學

這順序對我的工作完全 OK,但提醒:主視覺圖出來前,vocab cluster 一定要先鎖。否則畫面細節跟詞彙 mismatch — e.g. 圖裡有 elephant 但 vocab 沒教 elephant,user 學不到。流程上:

  1. vocab cluster 鎖(我做)
  2. 題目+互動寫完(陶做)
  3. 從 vocab + 題目推「該章畫什麼 object 細節」
  4. 視覺寫 prompt 餵 gpt-image-1
