## 1. Vision & 定位

**用養成一隻小怪的溫柔陪伴,包裝英檢練習 —— 你不逼牠,牠自己過得很好,還帶驚喜回來。**

- **類型**:虛擬寵物養成 + 怪物收集 + 英檢題庫。核心情緒延續拾光:**不焦慮、不懲罰、不催促**。
- **客群**:**全年齡**(YLE 兒童 + GEPT 初級青少年/成人)。怪物收集天生跨齡(像寶可夢)。
- **和拾光的關係**:sibling app,共用技術棧 + 美學語言 (Ghibli 暖色) + 題型引擎,但**世界觀獨立**(不是奶奶說故事,是你養的異世界小怪)。

## 3. 核心循環 (Core Loop)

```
練習英檢題 (一輪 N 題)
   → 依 first-try 答對率 給「成長值 + 獎勵」
       高答對率 = 成長快 + 掉好道具/服裝/紀念品
       低答對率 = 成長慢 + 普通掉落 (仍給, 不懲罰)
   → 成長值累積 → 升級 → 解鎖 (進化 / 服裝 / 新地點 / 故事章節)
   → (離線) 寵物自己出門冒險 (真實時間) → 回來帶 明信片/紀念品/日記/單字
   → 回家看寵物房 (可佈置) → 想再玩就再練
```

**「不讓寵物餓肚子」原則(作者硬規則)**:
- 寵物**永遠不會餓死 / 不會生病 / 不會離開**。
- 荒廢最壞的後果 = 只能吃「難吃的路邊飼料」(微委屈的表情 / 對話,**非紅色、非罪惡感、非死亡**)。
- 練習 = 給牠好料 / 帶乾糧走更遠,是「錦上添花」不是「保命」。
- 對照拾光 Decision Log:故事模式無 HP、force-correct + blindRetry,同一套「獎勵導向非懲罰」哲學。

## 4. 寵物系統

### 4.0 生物核心設定 (Tensura 史萊姆式)
- **咕嚕 = 一隻史萊姆基底的小怪**,吃「單字」吸收能量 → 長大、長出不同神話生物的特徵(角 / 翅膀 / 耳朵 / 眼睛…)→ 進化。
- 靈感 = 《轉生史萊姆》Rimuru「吃下生物吸收其能力/外型」的設定,對應我們「吃單字吸收知識」。
- 好處:史萊姆基底讓所有怪造型一致、生圖成本低(共用底 + 換特徵),又能「綜合好幾個神話的生物」(作者要的)、天然醜萌。

### 4.1 孵化 & 收集 (首發 v1 roster)
- 起始:1 顆蛋,練習到一定成長值 → 孵化 → 隨機異世界小怪(v1 池 5 隻,跨神話)。
- **首發 5 隻**(史萊姆基底 + 各一神話特徵,詳細生圖 prompt 見 creature-prompts 檔):
  1. **豆芽咕嚕 Sprout Gulu** — 頭頂冒新芽(成長象徵)· 綠 · 最單純,預設第一隻
  2. **狐火咕嚕 Kitsune Gulu** — 狐耳 + 飄浮小火焰(日本狐火)· 琥珀橙
  3. **龍鱗咕嚕 Dragon Gulu** — 小龍角 + 鱗片 + 龍鬚(東方龍)· 青碧
  4. **獨眼咕嚕 Cyclops Gulu** — 單顆大眼 + 小石棒(希臘獨眼巨人)· 灰陶,醜萌代表
  5. **羽蛇咕嚕 Quetzal Gulu** — 羽毛頸圈 + 小翅(中美洲羽蛇神)· 翠金
- 之後可獲得更多蛋(練習里程碑 / 冒險帶回 / IAP 特殊蛋),孵不同物種 → 圖鑑收集。
- 美術風格:Ghibli 暖色 + 異世界醜萌;外部 AI 生圖(比照拾光 mascot 流程,作者手動生)。**v1 只需 5 隻 + 蛋 + 每隻 1 個進化型**。

### 4.2 進化階段
- 每隻怪隨等級進化(幼體 → 成體 → …),外型變化。
- 進化型也要美術;v1 每隻先 2 階段。

### 4.3 換裝
- 解鎖帽子 / 眼鏡 / 衣服 / 配件,自由換裝。外觀 IAP 主力。
- 服裝要跨物種相容(或每物種一套)—— **待決:美術成本考量**。

## 5. 離家冒險 (旅行青蛙式)

- **真實時間制**:寵物離線時自己出門(出門數小時),回來帶戰利品。
- 帶回:①明信片+照片(各地 + 一句小故事)②紀念品/道具(佈置房間/換裝)③故事日記(拼世界觀)④旅途單字(掛英檢內容)。
- **推播**:寵物回家 / 有新明信片 → PWA push(溫柔、可關)。
- 練習加成:練得多 → 給牠帶乾糧 → 走更遠 / 帶更多回來(把「練習」和「冒險驚喜」正向掛鉤)。
- 參考先例:旅かえる(Travel Frog)、Neko Atsume、Finch。

## 6. 主畫面:寵物房

- 一間可佈置的小窩,寵物在裡面生活。
- 互動:點寵物(反應/對話)、餵食、換裝、看明信片牆 / 圖鑑 / 日記、**開始練習**。
- 佈置:冒險帶回的紀念品 / IAP 家具擺進房間。
- 導覽:房間(首頁)/ 圖鑑 / 冒險日記 / 練習 / 家長區(設定+變現)。

## 8. 變現

- **免費閘 = 一天 first-try 答錯 10 題**(2026-07-02 作者定案,可調常數 `DAILY_WRONG_CAP = 10`)。
  - **算 first-try 錯**(第一次就答錯),**不算 blindRetry 重試** —— blindRetry 一題可能錯多次,用總錯數不公平;first-try 才是真實答對率訊號(同拾光 `energy.ts`)。
  - 換算(錯 10 題 ≈ 能玩幾題):90% 答對率→~100 題、80%→~50、70%→~33、50%→20、25%(亂猜)→~13。最壞 ~13 題≈一節,**不另設保底**。
  - 設計哲學:**溫柔上限 + 獎勵準確度**,不是付費牆。答得好幾乎玩到飽;一直答錯 → 自然停損。
  - 主題包裝:錯一題 → 咕嚕打個小哈欠;錯滿 10 → 咕嚕睡著 😴「明天再一起學!」(家長可付費解鎖續玩)。
- **付費**:解除每日錯誤額度(續玩)+ 外觀 IAP(服裝 / 房間家具 / 特殊蛋外觀)。
- **⚠️ 已知取捨**:錯誤額度制對「準確用戶幾乎不撞牆」→ 閘本身催付費弱,**變現主力 = 外觀 IAP**。若日後要閘催轉換,再考慮加「題數硬上限」或訂閱制。
- **禁**:賭博式抽卡機率(學習 app 掠奪性 gacha 風險)、pay-to-win(付費不影響學習成效本身)。
- LTV 參考拾光模型(月費 × 訂閱月數;目標 LTV:CAC ≥ 3)。
